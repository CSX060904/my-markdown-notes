# Linux

## linux的基本常识

### linux网络连接三种方式

#### IP地址四段数字的含义

以 `192.168.100.99` 为例，这种写法叫**点分十进制**，代表一个32位的二进制数，分为4组，每组8位（0~255）。
要理解“网段”，必须结合**子网掩码**，最常见的是 `255.255.255.0`。

- **网络部分**：前三段数字（如 `192.168.100`）代表网络号，用于标识一个子网。
- **主机部分**：最后一段数字（如 `99`）代表该子网内的某一台主机。
- **同一网段**：指网络号相同，这样两台机器可以直接通信，无需路由器。
  例：`192.168.100.88/255.255.255.0` 和 `192.168.100.99/255.255.255.0` 在同一网段；
  而 `192.168.0.50/24` 和 `192.168.100.88/24` 不在同一网段

#### 桥接模式

下面，我们用一幅图来说明什么是桥接模式：

![image-20260601113359340](https://cdn.jsdelivr.net/gh/CSX060904/My-notes/img/image-20260601113359340.png)

**解释**：

在桥接模式下，虚拟机里面的网络地址必须和外部的网络地址保持在同一网段(也就是前三组数字必须是

一样的)。这样linux操作系统才可以和外部的机器进行通信。**但是如果用户人数太多，容易造成IP冲**

**突。**

**工作原理**
虚拟机直接连接到宿主机所在的物理网络，相当于网络上的一台独立设备。虚拟机会从物理路由器（或DHCP服务器）获取一个**与宿主机同网段**的真实局域网IP。

**网络通信**

- 虚拟机 ↔ 宿主机：通过物理交换机/路由器直接通信。
- 虚拟机 ↔ 局域网内其他设备：直接互通。
- 虚拟机 ↔ 外网：通过物理网关上网。

**IP配置要求**
虚拟机的IP必须和宿主机物理网卡**在同一个网段**，前三组数字相同（假设掩码为 `255.255.255.0`），且网关、DNS与宿主机一致。
例：宿主机IP `192.168.0.50/24`，虚拟机可设为 `192.168.0.100/24`。

**优点**

- 网络结构简单，虚拟机完全融入物理网络。
- 局域网内其他设备可直接访问虚拟机。

**缺点及IP冲突原因**

- 虚拟机会占用真实局域网的一个IP地址。
- **IP冲突风险高**：如果多个人同时使用桥接，且手动设置了相同的静态IP，或DHCP地址池不足，就会出现两台设备抢同一个IP，导致网络中断。
- 网络变动（如切换Wi-Fi）时虚拟机可能需重新获取IP。

#### NAT模式

![image-20260601113431191](https://cdn.jsdelivr.net/gh/CSX060904/My-notes/img/image-20260601113431191.png)

解释：

在NAT模式下，虚拟机里面的网络可以不再和主机里面的网络保持在同一网段。但是主机里面会存在虚

拟网卡(192.168.100.99)，这个虚拟网卡的ip地址必须和linux里面的IP地址(192.168.100.88)在同一网

段。这样linux就可以通过虚拟网卡和主机之间进行通信了。同时也可以通过主机的真实地址

(192.168.0.50)和外部网络进行通信。**这样的好处是可以避免造成IP冲突**。

**工作原理**
宿主机内部建立一个**独立的虚拟网络**，并虚拟出一块网卡（如VMnet8，IP `192.168.100.99`）。虚拟机连接在这个虚拟网络中，通过宿主机内置的NAT服务访问外网。

**网络通信**

- **虚拟机 ↔ 宿主机**：通过虚拟网卡 `192.168.100.99` 在私网内直接通信。
- **虚拟机 → 外网**：数据包先发送到宿主机的虚拟网卡，NAT服务把源地址换成宿主机物理网卡的真实IP（如 `192.168.0.50`），再发到外网。回程数据由NAT转回虚拟机。
- **外网设备无法主动访问虚拟机**（除非设置端口转发）。

**IP配置要求**
虚拟机IP只需要和**宿主机的虚拟网卡**在同一网段即可（例：虚拟网卡 `192.168.100.99`，虚拟机 `192.168.100.88`），**不需要和宿主机物理IP同网段**。网关通常指向虚拟网卡的IP。

**优点**

- **不占用物理网络的IP地址**，避免IP冲突。
- 即使宿主机切换网络环境，虚拟机网络配置一般不受影响。
- 多个虚拟机各自拥有私有IP，互相隔离，安全性好。

**为什么NAT模式不会造成IP冲突？**

- 虚拟机使用的是虚拟网络内部私有IP，**不出现在物理局域网中**。
- 外网只看到宿主机物理网卡IP，内部无论多少虚拟机、IP怎么配都不会和外部设备冲突。
- 多台宿主机的虚拟网络各自独立，即便虚拟机IP相同也互不干扰。

#### 主机模式

**工作原理**
宿主机生成一块专用虚拟网卡（如VMnet1，IP `192.168.56.1`），虚拟机连接到这个网卡上，形成一个**完全封闭的私有网络**。该网络既没有桥接到物理网卡，也没有NAT功能（默认）。

**网络通信**

- 虚拟机 ↔ 宿主机：通过虚拟网卡在同一网段内直接通信。
- **无法直接访问外网或局域网其他物理设备**（除非宿主机额外配置路由或共享连接）。

**IP配置要求**
虚拟机IP必须和宿主机上 Host-Only 虚拟网卡的IP**在同一网段**，例如虚拟网卡 `192.168.56.1/24`，虚拟机可设 `192.168.56.2/24`。

**适用场景**
需要宿主机与虚拟机私密通信、搭建隔离实验环境、不依赖外部网络的场合。

#### 总结

| 模式   | 虚拟机IP与谁同网段         | 能否访问外网 | 是否占用物理IP | IP冲突风险 | 适合场景             |
| :----- | :------------------------- | :----------- | :------------- | :--------- | :------------------- |
| 桥接   | 宿主机物理网卡             | 能           | 是             | 高         | 虚拟机需融入局域网   |
| NAT    | 宿主机虚拟网卡（如VMnet8） | 能           | 否             | 无         | 安全上网、避免IP冲突 |
| 仅主机 | 宿主机Host-Only虚拟网卡    | 否           | 否             | 无         | 隔离测试环境         |

## Linux 目录结构特点

- 采用**层级式树状结构**，根目录为 `/`
- 没有 Windows 中的盘符概念，所有文件和设备都从 `/` 开始
- **一切皆为文件**：硬件设备、进程信息等都以文件形式呈现

---

### 核心目录说明

| 目录     | 作用                                                   | 注意事项                                  |
| -------- | ------------------------------------------------------ | ----------------------------------------- |
| `/boot`  | 存放系统启动所需文件（内核、引导加载程序等）           | 损坏或删除将导致系统无法启动              |
| `/etc`   | 系统和大多数服务的配置文件（如 MySQL 的 `my.cnf`）     | 修改前建议备份                            |
| `/bin`   | 基础用户命令，如 `ls`, `cp`, `cat`                     | 现代系统中常是 `/usr/bin` 的链接          |
| `/sbin`  | 系统管理员用的管理命令，如 `fdisk`, `mkfs`             | 通常为 `/usr/sbin` 的链接                 |
| `/lib`   | 系统启动和基本命令所需的共享库（类似 Windows DLL）     | 删除会导致系统不可用，常链接到 `/usr/lib` |
| `/dev`   | 设备文件，硬件以文件形式存在（硬盘、终端等）           | 系统自动管理，勿随意删除                  |
| `/proc`  | 虚拟文件系统，内核与进程信息的映射                     | 勿随意修改，可能导致崩溃                  |
| `/sys`   | 内核设备、驱动等信息的虚拟文件系统                     | 同上                                      |
| `/srv`   | 服务启动后需要的数据                                   | 一般由服务程序维护，勿手动修改            |
| `/tmp`   | 临时文件存放，重启可能被清空                           | 所有用户可写，注意权限                    |
| `/home`  | 普通用户的个人主目录，如 `/home/tom`                   | 创建用户时自动生成，`userdel -r` 会删除   |
| `/root`  | root 用户的主目录                                      | 仅供超级用户使用                          |
| `/usr`   | 用户级别的应用程序、库、文档等，类似系统级的程序文件夹 | `/usr/local` 用于自行编译安装的软件       |
| `/opt`   | 第三方或附加软件的安装目录（如 JDK）                   | 可选，按需使用                            |
| `/media` | 自动挂载可移动设备（U盘、光盘）的挂载点                | 插入设备后系统自动在此创建目录并挂载      |
| `/mnt`   | 临时手动挂载外部存储的位置                             | 共享目录、额外硬盘等常挂载于此            |
| `/var`   | 频繁变化的数据：日志、缓存、运行时文件等               | `/var/log` 下存放各类日志                 |

## Linux的基本实操

### Linux vi/vim 编辑器

#### 简介

- 绝大多数 Linux 发行版都内置 `vi` 文本编辑器。
- `vim` 是 `vi` 的增强版，支持语法高亮、代码补全、编译及错误跳转等功能，尤其适合编程。

#### 三种工作模式

| 模式           | 说明                                                         |
| :------------- | :----------------------------------------------------------- |
| **正常模式**   | 打开文件即进入该模式。可以移动光标、删除字符/行、复制粘贴，但不能直接输入文字。 |
| **插入模式**   | 用于输入和编辑文本内容。按 `i`、`I`、`a`、`A`、`o`、`O` 等键进入，左下角显示 `-- INSERT --`。 |
| **命令行模式** | 执行保存、退出、查找、替换、显示行号等操作。在正常模式下输入 `:` 进入。 |

![image-20260603233125674](https://cdn.jsdelivr.net/gh/CSX060904/My-notes/img/image-20260603233125674.png)

#### 基本使用流程

1. **打开或创建文件**

   bash

   ```c++
   vim Hello.java
   ```

   

   自动进入**正常模式**。

2. **进入插入模式编辑内容**
   按 `i`（光标前插入），此时左下角出现 `INSERT`，即可输入代码或文字。

3. **保存并退出**

   - 按 `ESC` 返回正常模式
   - 输入 `:wq` 回车 → 保存并退出
   - 补充命令：
     - `:q` 直接退出（未修改时可用）
     - `:q!` 强制退出，不保存修改

#### 常用快捷键（均在正常模式下使用）

##### 复制与粘贴

- `yy` – 复制光标所在行
- `p` – 粘贴到光标下一行
- `2yy` – 复制从当前行开始的 2 行
- 示例：先按 `2yy` 再按 `p`，即完成“复制2行并粘贴”。

##### 删除

- `x` – 删除光标所在处字符
- `dd` – 删除光标所在行
- `2dd` – 删除当前行及下一行，共 2 行

##### 撤销

- `u` – 撤销上一步操作（可多次按）

##### 光标快速移动

- `gg` – 跳到文件开头
- `G` – 跳到文件末尾
- `5G` 或 `5 shift+g` – 跳到第 5 行

##### 搜索

- `/关键词` 回车 – 向下搜索关键词
- 按 `n` 查找下一个匹配项
- 示例：`/Hello` 再按 `n`

##### 行号设置

- `:set nu` – 显示行号
- `:set nonu` – 取消行号

##### 替换（命令行模式）

- `:s/World/shijie/` – 将**当前行**的第一个 `World` 替换为 `shijie`
- `:s/World/shijie/g` – 将**当前行**中所有 `World` 替换为 `shijie`
- `:%s/World/shijie/g` – 将**整个文件**中的所有 `World` 替换为 `shijie`

### Linux 关机与重启

#### 常用命令概览

| 命令       | 作用                                                         | 常用写法                   |
| :--------- | :----------------------------------------------------------- | :------------------------- |
| `shutdown` | 安全关闭或重启系统，可设置时间并广播消息                     | `shutdown -h now` 立即关机 |
| `halt`     | 立即停止系统，等价于 `shutdown -h now`（部分系统加 `-p` 断电） | `halt` 或 `halt -p`        |
| `reboot`   | 立即重启系统                                                 | `reboot`                   |
| `sync`     | 将内存缓冲区数据同步到磁盘（非关机命令）                     | `sync`                     |

#### 命令详解

##### shutdown（推荐日常使用）

- **立即关机**

  bash

  ```
  shutdown -h now
  ```

  

- **定时关机**

  bash

  ```
  shutdown -h +5            # 5分钟后关机
  shutdown -h 23:00         # 23:00 关机
  ```

  

- **立即重启**

  bash

  ```
  shutdown -r now
  ```

  

- **取消正在等待的关机**

  bash

  ```
  shutdown -c
  ```

  

##### halt

- 直接停止系统，不延时。部分系统需要添加 `-p` 才会切断电源：

  bash

  ```
  halt -p
  ```

  

- 与 `shutdown -h now` 类似，但 `shutdown` 会进行更完整的关闭流程（如通知用户、关闭服务）。

##### reboot

- 立即重启系统：

  bash

  ```
  reboot
  ```

  

- 也支持通过 `shutdown -r now` 实现重启。

##### sync（数据同步）

- 将内存中尚未写入磁盘的数据强制刷写到硬盘，避免数据丢失。
- **不是**关机命令，但经常在手动关机前执行（见注意事项）。

#### 注意事项

1. **现代关机命令已自动 sync**
   `shutdown`、`reboot`、`halt` 等命令在执行时都会自动调用 `sync`，正常关机无需手动执行 `sync`。
2. **特殊情况建议手动 sync**
   如果系统死机或进行强制关机（如直接切断电源、使用 `reboot -f` 等），无法保证数据被同步，此时建议在可能的情况下先执行 `sync`。
3. **远程连接时谨慎关机**
   服务器关机后将断开所有连接，确认无其他用户正在使用再执行关机或重启。

### Linux 用户登录与注销

#### 切换用户：su 命令

`su`（switch user）用于临时切换用户身份。

| 命令              | 作用                                | 说明                         |
| :---------------- | :---------------------------------- | :--------------------------- |
| `su root`         | 切换到 root 用户                    | 不改变环境变量，当前目录不变 |
| `su - root`       | 切换到 root，并加载 root 的完整环境 | 等同于登录 root              |
| `su - wen`        | 切换到普通用户 wen                  | 加载 wen 的环境              |
| `su` （不加参数） | 默认切换到 root                     | 等价于 `su root`             |
| `su -`            | 切换到 root 并加载完整环境          | **推荐**                     |

#### 密码验证规则

- **普通用户 → root**：需要输入 **root 用户的密码**。
- **root → 普通用户**：**不需要**密码，直接切换。
- **普通用户 → 另一个普通用户**：需要输入目标用户的密码。

#### 退出登录 / 返回原用户

| 命令            | 作用                             |
| :-------------- | :------------------------------- |
| `exit`          | 退出当前 shell                   |
| `logout`        | 在登录 shell 中注销（同 `exit`） |
| 快捷键 `Ctrl+D` | 效果等同于 `exit`                |

- 如果你通过 `su` 临时切换了用户，输入 `exit` 或 `Ctrl+D` 会**退回到之前的用户**，而不是注销系统。
- 如果你在最初的登录 shell 中执行 `exit`，则会**注销登录**，断开连接。

#### 操作示例

bash

```
# 当前用户为 wen（普通用户）
su - root            # 输入 root 密码，切换至 root 并获得完整环境
# 现在已经是 root 用户了
exit                 # 退回 wen 用户，或者 Ctrl+D

# 从 root 切换到 wen
su - wen             # 无需密码，直接切换
exit                 # 退回 root
```

#### 注意事项

- 尽量使用 `su -` 而不是裸 `su`，避免因环境变量缺失导致命令找不到或行为异常。
- 远程登录服务器时，完成工作后及时 `exit` 注销，避免留下未关闭的会话，既不安全也浪费资源。
- 如果提示 `su: Permission denied`，可能是目标用户 shell 被禁用或密码错误。

## Linux 用户管理

### 概述

- Linux 是多用户、多任务系统，每个使用者需拥有独立账号。
- 用户管理常用命令：`useradd`、`passwd`、`userdel`、`id`、`su`、`groupadd`、`groupdel`、`usermod`。

------

### 用户添加与家目录

**基本语法**

bash

```
useradd [选项] 用户名
```



**常见用法**

- 创建用户（自动在 `/home` 下生成同名家目录）：

  bash

  ```
  useradd kobe
  ```

  

- 创建用户并指定家目录：

  bash

  ```
  useradd -d /home/test king   # 家目录为 /home/test
  ```

  

**说明**

- 默认家目录 `/home/用户名`。
- 登录后直接进入自己的家目录。

------

### 设置与修改密码

bash

```
passwd 用户名
```



- 只有 root 可以替任意用户设置密码；普通用户只能修改自己的密码。
- 新密码若少于 8 个字符系统会警告，但仍可使用（视安全策略而定）。

------

### 删除用户

| 命令                | 作用                                   |
| :------------------ | :------------------------------------- |
| `userdel 用户名`    | 删除用户账号，保留家目录和邮件池       |
| `userdel -r 用户名` | 删除用户账号，并同时删除家目录及邮件池 |

> **注意**：仅删除账号后，原家目录属主会变为数字 UID，若该 UID 随后分配给新用户，目录会显示为新用户名。可事先备份再清理。

------

### 查询用户信息

- **查看用户 UID、GID 及所属组**

  bash

  ```
  id 用户名
  # 示例输出：uid=1001(kobe) gid=1001(kobe) groups=1001(kobe)
  ```

  

- **查看当前会话的用户身份**

  - `who am i`：显示登录时的原始用户（切换用户后结果不变）
  - `whoami`：显示当前有效用户（随 `su` 切换而改变）

------

### 切换用户

bash

```
su 用户名
```



- **root → 普通用户**：无需输入密码。
- **普通用户 → root**：需要输入 **root 的密码**。
- **普通用户 → 另一个普通用户**：需要输入 **目标用户的密码**。

> 建议使用 `su - 用户名` 来完整切换环境（加载目标用户的环境变量）。

------

### 用户组管理

#### 组的概念

- 组用于对多个用户进行统一的权限管理。
- 每个用户都有一个主组（创建用户时自动生成与用户名同名的组，或通过 `-g` 指定），还可拥有多个附加组。

#### 创建/删除组

bash

```
groupadd 组名      # 创建组
groupdel 组名      # 删除组（组内无用户时）
```



#### 创建用户时指定组

bash

```
useradd -g 组名 用户名
```



示例：

bash

```
groupadd gaibang
useradd -g gaibang guojing
id guojing   # gid 和 groups 均显示 gaibang
```



#### 修改用户的主组

bash

```
usermod -g 新组名 用户名
```



前提是新组必须已存在。

bash

```
groupadd taohuadao
usermod -g taohuadao guojing
```



> 若需要修改附加组，使用 `usermod -G 组名 用户名`（会覆盖原有附加组）。

------

### 与用户/组相关的重要文件

| 文件          | 用途                   | 关键字段格式                                                 |
| :------------ | :--------------------- | :----------------------------------------------------------- |
| `/etc/passwd` | 用户账户信息           | 用户名:密码占位符:UID:GID:描述:家目录:Shell                  |
| `/etc/shadow` | 用户加密密码及过期信息 | 用户名:加密密码:最近修改:最小天数:最大天数:警告:不活动:过期:保留 |
| `/etc/group`  | 组信息                 | 组名:密码占位符:GID:组内用户列表                             |

- 密码占位符通常为 `x`，真实密码存于 `/etc/shadow`。
- 若用户未设置密码，`shadow` 中密码字段常为 `!!` 或 `*`，表示账户被锁定无法直接登录。

## Linux 运行级别 (Runlevel)

### 概念

Linux 系统定义了多个**运行级别**，代表系统启动后所处的不同状态，用于控制启动哪些服务。

### 七种标准运行级别

| 级别  | 含义             | 说明                                           |
| :---- | :--------------- | :--------------------------------------------- |
| **0** | 关机             | 系统停止运行                                   |
| **1** | 单用户模式       | 仅 root 登录，无网络，常用于修复系统或找回密码 |
| **2** | 多用户（无网络） | 多用户但无 NFS 网络服务（极少使用）            |
| **3** | 完全多用户模式   | 命令行界面，有网络，**服务器常用**             |
| **4** | 保留             | 未使用，可由用户自定义                         |
| **5** | 图形界面         | X11 图形化登录，**桌面版常用**                 |
| **6** | 重启             | 系统重新启动                                   |

最常用的运行级别是 **3**（服务器）和 **5**（桌面）。

### 查看当前运行级别

bash

```
runlevel
# 输出示例：N 5 （上一个级别为 N 表示未知，当前为 5）
```

或

bash

```
who -r
```

### 切换运行级别

使用 `init` 命令（或 `telinit`）：

bash

```
init 3          # 切换到纯命令行模式
init 5          # 切换到图形界面
init 0          # 关机
init 6          # 重启
```

*在 systemd 系统中，`init` 命令会被转发给 `systemctl`，依然有效。*

### 设置系统默认的运行级别

#### CentOS 6 及更早版本

修改 `/etc/inittab` 中的 `id:5:initdefault:` 一行，将数字改为想要的级别。

#### CentOS 7 及更新版本（systemd）

不再使用 `/etc/inittab`，改用 `systemctl` 设置默认的 **target**：

bash

```
systemctl get-default                     # 查看当前默认目标
systemctl set-default multi-user.target   # 设置为命令行模式（对应级别3）
systemctl set-default graphical.target    # 设置为图形界面（对应级别5）
```



这些 target 对应关系：

- `multi-user.target` ↔ 运行级别 3
- `graphical.target` ↔ 运行级别 5
- `poweroff.target` ↔ 运行级别 0
- `rescue.target` ↔ 单用户模式（类似级别1）
- `reboot.target` ↔ 运行级别 6

### 注意事项

- 切换运行级别会终止当前未在目标级别下运行的服务，可能导致未保存的工作丢失，切换前请保存。
- 远程服务器切勿随意切换到运行级别 0 或 6，会导致断网或重启。
- 初学阶段建议使用带有图形界面的运行级别（5），方便操作。

## 如何找回 root 密码

（CentOS 7/8 单用户模式）

**适用场景**：忘记 root 密码，且可以物理接触服务器或通过控制台访问。

### 操作步骤

1. **重启系统，进入 GRUB 菜单**
   开机后在 GRUB2 引导界面按 `e` 键进入编辑模式。

   > 若菜单被隐藏，可在启动时反复按 `Esc` 或 `Shift` 调出菜单。

2. **编辑内核启动参数**
   用方向键找到以 `linux16` 或 `linuxefi` 开头的行，在行末追加：

   text

   ```
   init=/bin/sh
   ```

   确保与前面内容用空格隔开。

3. **进入单用户 Shell**
   按 `Ctrl+x` 或 `F10` 启动系统，此时会直接进入一个 **root 权限的 shell**（无需密码）。

4. **重新挂载根文件系统为读写**
   当前根文件系统默认为只读（ro），需重新挂载：

   bash

   ```
   mount -o remount,rw /
   ```

5. **修改 root 密码**
   执行：

   bash

   ```
   passwd
   ```

   按提示输入新密码并确认（虽然最短可设1位，但建议8位以上）。

6. **处理 SELinux 上下文（必须）**
   由于采用了 `init=/bin/sh` 方式，重启后 SELinux 可能禁止登录，需创建标记文件让系统在启动时自动重新标记：

   bash

   ```
   touch /.autorelabel
   ```

7. **重启系统**
   执行以下命令让系统正常启动：

   bash

   ```
   exec /sbin/init
   ```

   

   系统会自动重启，并开始 SELinux 重新标记（可能需要几分钟），完成后即可用新密码登录。

### 替代方法

在步骤2中，不写 `init=/bin/sh`，改为在 `linux16` 行末追加：

text

```
rd.break enforcing=0
```



然后按 `Ctrl+x` 启动，进入紧急模式后执行：

bash

```
mount -o remount,rw /sysroot
chroot /sysroot
passwd
touch /.autorelabel        # 仍然建议执行
exit
exit
```



系统重启后同样生效，且 `enforcing=0` 临时关闭 SELinux 可避免登录问题。

### 注意事项

- 此方法需要物理访问服务器或通过虚拟化控制台操作，**远程 SSH 无法使用**。
- 若 GRUB 设置了密码保护，需先输入 GRUB 密码才能编辑启动项。
- 修改完成后，**务必测试新密码**并确保 SELinux 上下文正确。

## Linux 帮助指令

### man 命令

`man` 用于查看系统命令、配置文件、函数等的详细手册页。

**语法**

bash

```
man [选项] 命令名
```



**示例：查看 `ls` 命令的手册**

bash

```
man ls
```



在手册中可以使用以下操作：

- `↑` `↓` 或 `j` `k` 滚动
- `空格` 向下翻页，`b` 向上翻页
- `/关键词` 向下搜索，`n` 查找下一个
- `q` 退出

#### 常用 ls 参数速查

- `ls -a`：列出所有文件，包括隐藏文件（以 `.` 开头）。
- `ls -l`：以长列表格式显示权限、所有者、大小、时间等信息。
- `ls -la` 或 `ls -al`：长列表显示所有文件（含隐藏文件）。
- `ll`：通常为 `ls -l` 的别名（部分系统自带）。
- `ls -lh`：以人类可读的大小格式显示（如 `1K` `234M`）。

参数可组合，例如：

bash

```
ls -lha
```



### help 命令

`help` 用于显示 **bash shell 内置命令** 的帮助信息，因为这些命令不能用 `man` 查找（`man cd` 查不到，需用 `help cd`）。

**语法**

bash

```
help 内置命令
```



**示例**

bash

```
help cd
help for
```



也可以直接输入 `help` 列出所有内置命令。

### 其他快速帮助方式

| 方式           | 说明                                | 示例         |
| :------------- | :---------------------------------- | :----------- |
| `命令 --help`  | 多数外部命令支持，显示简要用法      | `ls --help`  |
| `info 命令`    | GNU 格式的详细文档，比 man 更结构化 | `info ls`    |
| `whatis 命令`  | 显示一行简短描述                    | `whatis ls`  |
| `whereis 命令` | 查找命令二进制、源码和手册位置      | `whereis ls` |

### 总结

- 需要**完整说明和所有选项** → `man 命令`
- 快速查看**常用语法** → `命令 --help`
- 确认是否为 **bash 内置命令** → `help 命令`
- 需要**教程式导航** → `info 命令`
