# 数据结构与算法

## 栈

### 单调栈

#### 定义
单调栈是指栈内元素（通常存的是数组的**下标**，但也可以是元素值）始终保持单调递增或单调递减的栈。 
通过暴力弹出破坏单调性的元素，保证栈的有序性，从而能在 O(1) 时间内获得某个方向上第一个满足特定大小关系的元素。

#### 能解决的问题
- 寻找一个序列中每个元素 **左侧/右侧 第一个比它大/小** 的元素（或其位置）。
- 计算以某个值为最值的连续区间范围（如柱状图中的最大矩形）。
- 维护动态区间最值关系，用来优化 DP 等。

#### 核心思想
**「及时删除永远不会被用到的候选元素」**。 
以"向右找第一个比自己大的元素"为例，从左向右扫描（或反向扫描），对于当前元素，栈里存的是左侧/右侧待匹配的元素。如果当前元素更大，那么栈中那些更小的元素就再也不可能成为后续元素的"第一个更大数"，因为当前元素不仅更大，而且更近。于是可以安全弹出它们并记录答案。

通过这样的动态维护，栈始终保持单调，每次查询答案就是栈顶元素，时间复杂度 O(n)。

#### 模板（右侧第一个更大，从右向左遍历）
你这段代码要找的是：对于每个位置 `i`，它的**右侧第一个大于 `a[i]` 的元素的位置**，如果不存在则为 0。

```cpp
// 模板：找每个元素右侧第一个比它大的元素的下标（1-indexed）
vector<int> nextGreaterRight(vector<int>& a) {
    int n = a.size() - 1;          // a[1..n]
    stack<int> st;                 // 存下标，单调递减栈（栈底到栈顶的值递减）
    vector<int> ans(n + 1, 0);
    for (int i = n; i >= 1; --i) {
        // 当前元素 a[i] 比栈顶元素大，栈顶元素不可能成为 i 左边的元素的答案
        while (!st.empty() && a[st.top()] <= a[i]) {
            st.pop();
        }
        // 此时栈顶就是右侧第一个更大的（若栈空则没有）
        ans[i] = st.empty() ? 0 : st.top();
        st.push(i);
    }
    return ans;
}
```

**为什么是递减栈？** 
栈里存的是还未找到"右侧更大元素"的下标。当遇到一个新元素且它更大时，它会"终结"栈中那些比它小的元素的等待——因为对它们来说，新元素就是它们右边第一个更大的数。处理完后，新元素继续等待它右边的更大数，入栈后仍保持栈内对应值递减。因此栈是**严格递减**的（值从大到小）。

**遍历方向的影响**：  
- 从右向左：边遍历边"结算"，即处理当前元素时，栈里都是它右边的元素，可直接得出答案。  
- 也可以从左向右：栈里存尚未找到答案的下标，每次遇到一个数就作为"更大元素"去结算栈中比它小的数。两种方向效果一样。

#### 四种标准模板（存下标，均可根据需要改为存值）
设数组 `a` 下标从 `1` 到 `n`，默认 0 表示没有找到。

5.1 右侧第一个更大

- 栈：单调递减（值递减，栈顶最小）
- 遍历：从右向左（或从左向右都可）
```cpp
for (int i = n; i >= 1; --i) {
    while (!st.empty() && a[st.top()] <= a[i]) st.pop();
    ans[i] = st.empty() ? 0 : st.top();
    st.push(i);
}
```

右侧第一个更小

- 栈：单调递增（值递增，栈顶最大）
- 条件：`a[st.top()] >= a[i]` 时弹出
```cpp
for (int i = n; i >= 1; --i) {
    while (!st.empty() && a[st.top()] >= a[i]) st.pop();
    ans[i] = st.empty() ? 0 : st.top();
    st.push(i);
}
```

左侧第一个更大（从右向左同理，换方向即可）

- 栈：单调递减（值递减，栈顶最小）
- 遍历：从左向右
```cpp
for (int i = 1; i <= n; ++i) {
    while (!st.empty() && a[st.top()] <= a[i]) st.pop();
    ans[i] = st.empty() ? 0 : st.top();
    st.push(i);
}
```

左侧第一个更小

- 栈：单调递增（值递增，栈顶最大）
- 遍历：从左向右
```cpp
for (int i = 1; i <= n; ++i) {
    while (!st.empty() && a[st.top()] >= a[i]) st.pop();
    ans[i] = st.empty() ? 0 : st.top();
    st.push(i);
}
```

**记忆口诀**：  
- **找更大**：维护**递减栈**（让更大值能保留下来，小数被挤掉）  
- **找更小**：维护**递增栈**（让更小值保留下来，大数被挤掉）  
- 遍历方向决定是找左侧还是右侧。

#### 边界条件与等号处理

条件中的 `<=` 还是 `<` 根据题意而定：  
- `<=`：**严格更大/更小**。相等的不算，会被弹出，即将相等的元素视作"不大于/不小于"自己。  
- `<`：**非严格更大/更小**。相等的会保留，即第一个大于等于/小于等于的元素。

#### 复杂度分析
- 每个元素最多入栈一次、出栈一次，总时间复杂度 **O(n)**。
- 空间复杂度 **O(n)**（栈与答案数组）。

#### 正确性简要证明
以右侧第一个更大为例：对于位置 `i`，当处理到它时，栈内存在所有下标 `j > i` 且满足对于任意 `k` 在 `(i, j)` 之间，`a[k] ≤ a[i]`。  
- 若栈非空，栈顶就是因为比 `a[i]` 大而没有被弹出的最近元素，正是右侧第一个更大。  
- 弹出操作：那些 `a[st.top()] ≤ a[i]` 的元素，对于 `i` 左侧的任意位置 `l < i`，如果它们比 `a[l]` 大，那么 `a[i]` 也一定比 `a[l]` 大且更近，因此这些元素永远不会成为 `l` 的答案，可以放心丢掉。

#### 常见应用与变体
1. **下一个更大元素 I/II**（LeetCode 496, 503）  
   - 循环数组：将原数组复制一遍接在后面，遍历 2n 个元素，最后取前 n 个结果。
2. **每日温度**（LeetCode 739）  
   - 求的是间距而不是下标，将答案改为 `ans[i] = st.empty() ? 0 : st.top() - i;`
3. **柱状图中的最大矩形**（LeetCode 84）  
   - 找每个高度左右第一个比它小的位置，确定以该高度为高的最大宽度。
4. **接雨水**（LeetCode 42）  
   - 可用单调栈维护递减的柱子，遇到更高的柱子时形成凹槽积水。
5. **表达式求值/括号匹配**  
   - 单调栈思想也出现在一些优先级比较的题目中。

#### 代码实现的一些细节
- 栈内通常存下标，方便计算距离，也能直接取值 `a[st.top()]`。
- `ans` 初始化为 0（题中无满足条件时的返回值）。
- 若题目要求返回的是元素值而非下标，改成 `ans[i] = st.empty() ? -1 : a[st.top()];` 即可。
- 使用 `ios::sync_with_stdio(false); cin.tie(nullptr);` 加快读入，防止大数据超时。

## 树

### 概念/术语



### DFS序/DFN

在算法竞赛和数据结构中，**DFS 序**和 **dfn** 经常一起出现，它们的关系可以这样理解：

- **DFS 序**：对树进行深度优先搜索时，**节点的访问顺序序列**（通常指第一次进入节点时的顺序）。
- **dfn**（depth-first number）：即“深度优先数”，是 **DFS 序中每个节点被赋予的序号（时间戳）**。

简单来说：`dfn` 是一个数组，`dfn[u]` 表示节点 `u` 在 DFS 序中是第几个被访问的。**平时大家说的“求一棵树的 DFS 序”，本质上就是求出整棵树的 `dfn` 值**，所以有时这两个词会被混用。

---

#### 生成 dfn（DFS 序）

从根开始深搜，每**第一次**进入一个节点时，就给它分配一个递增的编号：

```cpp
int timer = 0;
int dfn[N], sz[N];

void dfs(int u, int fa) {
    dfn[u] = ++timer;   // 进入 u，分配时间戳
    sz[u] = 1;
    for (int v : edges[u]) {
        if (v == fa) continue;
        dfs(v, u);
        sz[u] += sz[v]; // 统计子树大小
    }
}
```

执行后：
- `dfn` 数组就是节点的 **DFS 序编号**（从 1 开始）。
- 如果按 `dfn` 从小到大把节点排成一列，这个序列就叫 **DFS 序**。

---

#### 最重要的性质：子树 = 连续区间

在 `dfn` 序列中，**同一子树内的所有节点，其 `dfn` 值一定构成一段连续的区间**：
- 左端点为 `dfn[u]`
- 右端点为 `dfn[u] + sz[u] - 1`

这让我们可以把“子树修改/子树查询”转换成**区间操作**，配合树状数组、线段树在 O(log n) 时间内完成。

**举例：**
一棵树：
```
    1
   / \
  2   3
 / \
4   5
```
DFS 顺序（先根，再左子树，再右子树）：1 → 2 → 4 → 5 → 3。 
则：

| 节点 | dfn  | sz   | 子树区间 |
| ---- | ---- | ---- | -------- |
| 1    | 1    | 5    | [1, 5]   |
| 2    | 2    | 3    | [2, 4]   |
| 3    | 5    | 1    | [5, 5]   |
| 4    | 3    | 1    | [3, 3]   |
| 5    | 4    | 1    | [4, 4]   |

节点 2 的子树包含 2、4、5，其 `dfn` 正好是 2、3、4，连续的一段。

---

#### 易混淆概念：欧拉序（括号序）

有时“DFS 序”也可能指**欧拉序**（Euler Tour），它记录**进入和离开**两个时间戳：
- `in[u]`：进入时间
- `out[u]`：离开时间

这样每个节点出现两次，总长度为 2n。欧拉序处理**路径问题**更方便，而本文说的 `dfn + size` 更适合**子树问题**。在中文算法圈里，**不加前缀的“DFS 序”绝大多数情况就指我们上面说的 `dfn` 序**。

---

#### 总结

- **DFS 序**：深度优先遍历的节点访问顺序（序列）。
- **dfn**：每个节点在这个顺序中的**具体序号**（时间戳）。
- 核心推导：`dfn` 配合子树大小 `sz`，可以将子树操作变为 O(log n) 的区间操作，是树问题转线性结构的常用技巧。

## 字符串

### Manacher算法

#### 目标
在线性时间 O(n) 内找出字符串的最长回文子串长度。

#### 字符串预处理
**目的**：统一处理奇数、偶数长度的回文，并加哨兵避免越界。

```cpp
string t;
t.push_back('$');          // 首哨兵
t.push_back('#');
for(int i = 0; i < len; i++){
    t.push_back(s[i]);
    t.push_back('#');
}
t.push_back('^');          // 尾哨兵
```

- 原串 `s` 的每个字符前后都加上 `#`。
- 最前面放一个不可能出现的字符（如 `$`），最后面放另一个（如 `^`），这样在中心扩展时永远不会越界。

变换示例： 
`s = "aba"` → `t = "$#a#b#a#^"` 
`s = "abba"` → `t = "$#a#b#b#a#^"`

> 使用 `$` 和 `^` 是常见习惯，只要保证它们不会出现在原串，且互不相等即可。

#### 核心数组与变量
```cpp
int len = t.length();
vector<int> d(len, 1);   // d[i] 记录以 t[i] 为中心的回文半径（包含中心）
d[0] = 0;                // 哨兵位置不用
d[1] = 1;                // 第一个 '#' 半径至少为 1
int l = 1, r = 1;        // 当前已知最右回文子串的左右边界（闭区间）
int ans = 0;             // 记录最大 d[i]
```

- `d[i]`：回文半径，例如 `"#a#"` 的中心 `'a'` 的 `d = 2`（`# a #` 共 3 个字符，中心算半径 2）。
- `l, r`：当前能到达最右侧的回文串的**左边界和右边界**。
- `ans`：记录过程中最大的半径。

#### 核心循环：利用对称性加速
```cpp
for(int i = 2; i < len; i++) {               // 从 2 开始，跳过首哨兵
    if(i <= r)
        d[i] = min(d[r - i + l], r - i + 1); // 用对称点信息初始化半径
    while(t[i - d[i]] == t[i + d[i]])
        d[i]++;                              // 朴素扩展
    if(i + d[i] - 1 > r){
        l = i - d[i] + 1;                    // 更新左边界
        r = i + d[i] - 1;                    // 更新右边界
    }
    ans = max(ans, d[i]);
}
```

对称性复用

- 当 `i` 还在当前最右回文 `[l, r]` 内部时，它关于中心的镜像点下标为 `r - i + l`（即 `l + r - i`）。
- 该镜像点的半径 `d[r - i + l]` 已经在前面算出来。
- 但镜像半径可能超出区间，所以取 `min(镜像半径, r - i + 1)` 作为初值（`r - i + 1` 是从 `i` 到右边界 `r` 的最远距离）。
- 如果 `i > r`，说明完全在已知回文外，跳过初始化，`d[i]` 保持默认值 `1`。

中心扩展

- 从 `d[i]` 的初值开始，尝试向两边比较字符：
  ```cpp
  while(t[i - d[i]] == t[i + d[i]]) d[i]++;
  ```
- 因为首尾有哨兵，当扩展到边界时，必然不等，循环自动停止，无需额外判断。

更新最右边界

- 如果以 `i` 为中心的回文**右边界** `i + d[i] - 1` 超过了原来的 `r`，就更新 `l` 和 `r`，这样后面的 `i` 就能获得更大的复用机会。

最终答案

```cpp
cout << ans - 1 << endl;
```
- `d[i]` 是变换串中的半径，它比真正的回文长度多 1（中心本身多算了一个 `#` 或字符的重复计数）。
- 例如 `"a"` 变换为 `"$#a#^"`，半径 `d[2]=2`，真实长度为 `1`。
- 所以最长回文长度 = `最大半径 - 1`。

#### 完整代码
```cpp
#include <bits/stdc++.h>
using namespace std;

void solve() {
    string s;
    cin >> s;
    int len = s.length();
    string t;
    t.push_back('$');
    t.push_back('#');
    for(int i = 0; i < len; i++){
        t.push_back(s[i]);
        t.push_back('#');
    }
    t.push_back('^');
    len = t.length();
    vector<int> d(len, 1);
    d[0] = 0;
    d[1] = 1;
    int l = 1, r = 1;
    int ans = 0;
    for(int i = 2; i < len; i++){          // 实际使用建议 i < len - 1，看注意事项
        if(i <= r)
            d[i] = min(d[r - i + l], r - i + 1);
        while(t[i - d[i]] == t[i + d[i]])
            d[i]++;
        if(i + d[i] - 1 > r){
            l = i - d[i] + 1;
            r = i + d[i] - 1;
        }
        ans = max(ans, d[i]);
    }
    cout << ans - 1 << endl;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    solve();
    return 0;
}
```

#### 注意事项
1. **循环边界**：代码中 `for(int i = 2; i < len; i++)` 会访问到末尾哨兵 `'^'`（下标 `len-1`）。  
   虽然 `while` 扩展时会因首尾哨兵而停止，但最好让循环 `i < len - 1`，这样彻底避免对哨兵进行无意义的半径计算，也更符合逻辑（哨兵不是真实回文中心）。
2. **哨兵选择**：`'$'` 和 `'^'` 不能出现在原字符串中，且两者不能相同。如果原串可能包含这些字符，请更换为其他不出现的字符，或者使用 `'\0'` 结合边界检查（但哨兵法最简洁）。
3. **结果必须 `-1`**：这是很多新手容易忘记的地方，输出 `ans` 会多算一个字符。
4. **时间复杂度 O(n)**：虽然有两层循环，但内层 `while` 只在右边界 `r` 需要向右推进时才真正执行，`r` 从 1 增加到 len，总比较次数是线性的。

## 排序

### 基本概念

#### 就地排序

- **定义**：排序过程中仅使用 **常数级别（O(1)）** 的额外空间，直接修改原数组。
- **特点**：节省内存，但可能不适合稳定性或特定数据分布。
- **例子**：
  - ✅ 就地排序：插入排序、选择排序、堆排序、快速排序（通常视为就地，实际栈空间 O(log n)）
  - ❌ 非就地排序：归并排序（经典实现需 O(n) 辅助空间）、计数排序（需额外计数数组）

#### 内部排序 vs 外部排序

| 类型     | 条件                           | 典型算法                           | 存储介质                  |
| :------- | :----------------------------- | :--------------------------------- | :------------------------ |
| 内部排序 | 所有记录可一次载入内存         | 快速排序、堆排序、归并排序等       | 内存                      |
| 外部排序 | 记录数量太大，无法一次载入内存 | 多路归并排序、败者树、置换选择排序 | 内存 + 磁盘（或其他外存） |

- **关键**：外部排序需要**读写外部存储**，通常采用“多路归并”策略，将大文件分成多个小块排序后再合并。

#### 稳定排序

- **定义**：如果待排序序列中存在多个**键值相同**的记录，排序后它们的**相对顺序保持不变**。
- **重要性**：当需要按多个字段排序（例如先按姓名排序，再按年龄排序）或处理键值对数据时，稳定性保证结果正确。
- **例子**：
  - ✅ 稳定排序：冒泡排序、插入排序、归并排序、计数排序、基数排序
  - ❌ 不稳定排序：选择排序、快速排序（常规实现）、堆排序

### 直接插入排序

#### 基本思想

将待排序序列分为 **有序区** 和 **无序区**。初始时有序区只有第一个元素，然后依次从无序区取出第一个元素，将它插入到有序区的正确位置，使有序区长度逐渐增加，直到全部有序。

#### 算法特点

- **就地排序**：只使用常数个额外空间（`x`, `j` 等），在原数组上修改。
- **稳定排序**：当遇到相等元素时，不会交换位置，相对顺序不变。
- **时间复杂度**：
  - 最好情况（已升序）：O(n)
  - 最坏情况（降序）：O(n²)
  - 平均：O(n²)

#### 逻辑及代码实现

```c
#include <stdio.h>
int n, a[105];

int main() {
    scanf("%d", &n);
    for (int i = 1; i <= n; i++)
        scanf("%d", &a[i]);

    int x, j;
    // 共进行 n-1 趟插入，每趟处理无序区的第一个元素
    for (int i = 1; i <= n - 1; i++) {
        // 第 i 趟时，有序区为 a[1] ~ a[i]，无序区为 a[i+1] ~ a[n]
        x = a[i + 1];          // 取出无序区第一个元素（待插入元素）

        // 在有序区中倒着查找插入位置，同时将大于 x 的元素后移
        for (j = i; j >= 1; j--) {
            if (a[j] > x) {
                a[j + 1] = a[j];   // 大于 x 的元素向后移动一位
            } else {
                break;             // 找到第一个 ≤ x 的位置，停止移动
            }
        }
        // 循环结束后，j 指向第一个 ≤ x 的元素（或 j == 0）
        // x 应该放在 j+1 的位置
        a[j + 1] = x;
    }

    for (int i = 1; i <= n; i++)
        printf("%d ", a[i]);
    return 0;
}
```

#### 示例

| 初始      | 有序区    | 无序区    | 操作                                                         |
| :-------- | :-------- | :-------- | :----------------------------------------------------------- |
| 3 1 7 5 2 | [3]       | [1,7,5,2] | 取 x=1，与3比较，3>1后移→[ ,3]；j=0，插入→[1,3]              |
| 1 3 7 5 2 | [1,3]     | [7,5,2]   | 取 x=7，与3比较，3<7不后移，直接插在3后面→[1,3,7]            |
| 1 3 7 5 2 | [1,3,7]   | [5,2]     | 取 x=5，倒着比较：7>5后移→[1,3, ,7]；3<5停止，j指向3，插在3后面→[1,3,5,7] |
| 1 3 5 7 2 | [1,3,5,7] | [2]       | 取 x=2，倒着比较：7>2后移→[1,3,5, ,7]；5>2后移→[1,3, ,5,7]；3>2后移→[1, ,3,5,7]；1<2停止，插在1后面→[1,2,3,5,7] |

------

### 插入排序的两种优化

#### 折半插入排序

- **思想**：在有序区查找插入位置时，使用 **折半查找（二分查找）**，减少比较次数。
- **移动次数不变**：仍需移动元素腾出空位，时间复杂度仍为 O(n²)，但比较次数从 O(n²) 降为 O(n log n)。
- **稳定性**：稳定。

#### 2-路插入排序

- **思想**：借助一个环形辅助数组，减少移动次数。将第一个元素作为基准，比它大的放右侧，比它小的放左侧，最终将环形数组线性化。
- **优点**：移动次数约为 n²/8，优于直接插入排序。
- **缺点**：需要额外 O(n) 空间，且实现较复杂。

------

### 希尔排序

#### 基本思想

希尔排序（缩小增量排序）是直接插入排序的改进。它先将整个序列按一定 **增量 d** 分成若干子序列，分别对每个子序列进行直接插入排序；然后逐渐缩小增量，重复上述过程；当增量减至 1 时，整个序列基本有序，再执行一次直接插入排序即可。

#### 算法特点

- **就地排序**：在原数组上修改。
- **不稳定排序**：因为相同元素可能被分到不同子序列中，相对顺序可能改变。
- **时间复杂度**：
  - 最坏情况：O(n²)（取决于增量序列）
  - 平均情况（希尔增量）：约 O(n^1.3) ~ O(n^1.5)
- **常用增量序列**：`d = n/2, d = d/2, ... , 1`（希尔原始序列）

#### 逻辑及代码实现

```c
#include <stdio.h>
int n, a[105];

int main() {
    scanf("%d", &n);
    for (int i = 1; i <= n; i++)
        scanf("%d", &a[i]);

    int k = 0, x, j;
    // 增量 d 从 n/2 开始，每次减半，直到 d = 1
    for (int d = n / 2; d >= 1; d = d / 2) {
        k++;   // 记录趟数（仅用于打印）
        // 对每个子序列进行直接插入排序
        // 注意：i 从 1+d 开始，即每个子序列的第二个元素（因为子序列第一个元素是 a[1], a[2], ..., a[d]）
        for (int i = 1 + d; i <= n; i++) {
            x = a[i];                     // 待插入的元素
            // 在本子序列中，元素下标为 i, i-d, i-2d, ...
            // 从当前位置的前一个同组元素开始往前找插入位置，并后移
            for (j = i - d; j >= 1; j -= d) {
                if (a[j] > x) {
                    a[j + d] = a[j];      // 大于 x 的元素向后移动 d 位
                } else {
                    break;                // 找到合适位置，停止移动
                }
            }
            // 插入 x 到正确位置（j+d）
            a[j + d] = x;
        }
        // 打印每一趟结果（可省略）
        printf("第%d趟排序的结果: ", k);
        for (int i = 1; i <= n; i++)
            printf("%d ", a[i]);
        printf("\n");
    }

    return 0;
}
```

#### 示例

对 {49,38,65,97,76,13,27,49,55,4} 进行希尔排序（n=10）

- **第一趟**：d = 10/2 = 5
  子序列1：下标 1,6 → {49,13} → 排序后 {13,49}
  子序列2：下标 2,7 → {38,27} → {27,38}
  子序列3：下标 3,8 → {65,49} → {49,65}
  子序列4：下标 4,9 → {97,55} → {55,97}
  子序列5：下标 5,10 → {76,4} → {4,76}
  结果：13 27 49 55 4 49 38 65 97 76
- **第二趟**：d = 5/2 = 2
  子序列1：下标 1,3,5,7,9 → {13,49,4,38,97} → 插入排序后 {4,13,38,49,97}
  子序列2：下标 2,4,6,8,10 → {27,55,49,65,76} → 插入排序后 {27,49,55,65,76}
  结果：4 27 13 49 38 55 49 65 97 76
- **第三趟**：d = 2/2 = 1
  对整个序列进行直接插入排序（此时序列已基本有序）
  最终结果：4 13 27 38 49 49 55 65 76 97

#### 关键理解

- **增量 d 同时也是组数**：例如 d=5 表示有 5 组，每组约 n/d 个元素（不一定严格相等）。
- **内层循环 `for(i=1+d; ...)`**：为什么从 `1+d` 开始？
  因为每组中第一个元素（下标 1…d）已经是有序的（单元素自然有序），所以我们从每组的第二个元素开始执行插入排序。
- **跳跃式移动**：元素移动的步长是 `d`，而不是 1，这使得小元素能快速跳到前面，减少最后 d=1 时的移动次数。

### 总结

| 排序算法     | 是否就地        | 是否稳定 | 时间复杂度（平均）    | 特点                       |
| :----------- | :-------------- | :------- | :-------------------- | :------------------------- |
| 直接插入排序 | ✅               | ✅        | O(n²)                 | 数据量小或基本有序时效率高 |
| 折半插入排序 | ✅               | ✅        | O(n²)（比较次数减少） | 查找位置用二分             |
| 2-路插入排序 | ❌（需辅助数组） | ✅        | O(n²)（移动次数减少） | 环形数组减少移动           |
| 希尔排序     | ✅               | ❌        | O(n^1.3)~O(n²)        | 分组跳跃插入，效率高       |

## 计算几何

### 扫描线

#### 什么是扫描线？
- **一句话定义**：把静态的区间/线段问题，转化为“事件点”的排序，然后像扫描一样从左到右处理这些事件，边扫描边维护当前状态，最终得到答案。
- **本质**：差分 + 排序 + 遍历。

#### 核心思想
- 原来需要枚举所有可能的坐标点（范围可能 1e9），但真正使状态发生变化的点只有区间端点。
- 每个区间可以拆成两个“事件”：
  - 左端点：开始覆盖，影响 +1
  - 右端点+1（闭区间时）：结束覆盖，影响 -1
- 把所有事件按坐标排序，依次处理，用一个计数器累加影响值，计数器的即时值就是当前位置的覆盖数。

#### 识别特征（什么时候用扫描线）
1. **问题描述有“区间”**，且问 **“最多有多少个区间同时重叠”** 或类似的最大值。
2. **需要选择一个位置/长度固定的区间**，问最多能覆盖多少个给定区间（可以转化为某个参数的覆盖区间重叠问题）。
3. **数据范围**：`n` 很大 (1e5+)，坐标范围也很大 (1e9)，不能用数组直接模拟坐标。
4. **矩形面积并、周长并** 等二维问题，会用到扫描线+线段树（进阶版）。

#### 通用解题步骤
1. **转化条件**：将问题转化为：求某个参数 `x` 被多少个区间覆盖。
   - 比如：选择一个 `[L, L+k]` 与 `[l_i, r_i]` 相交 → 推导出 `L` 必须满足 `l_i - k ≤ L ≤ r_i`，所以每个区间对应一个对 `L` 的限制区间 `[l_i-k, r_i]`。
2. **构造事件**：
   - 每个区间 `[A, B]` 生成两个事件：
     - `(A, +1)`  进入
     - `(B + 1, -1)` 离开（注意是 `B+1`，因为闭区间）
3. **事件排序**：
   - 按坐标从小到大排序。
   - 如果坐标相同，通常让 `+1` 排在 `-1` 前面（这样能正确记录最大值，避免提前减少）。
4. **扫描处理**：
   - 初始化 `cur = 0, ans = 0`
   - 遍历排序后的事件：`cur += delta`，`ans = max(ans, cur)`
5. **输出** `ans`。

#### 代码模板（C++）
```cpp
#include <bits/stdc++.h>
using namespace std;
typedef long long ll;

int main() {
    int n;
    cin >> n;
    vector<pair<ll, int>> events;
    for (int i = 0; i < n; i++) {
        ll A, B;
        cin >> A >> B;   // 每个区间 [A, B]
        events.push_back({A, 1});
        events.push_back({B + 1, -1});   // B+1 离开
    }
    sort(events.begin(), events.end());

    int cur = 0, ans = 0;
    for (auto &e : events) {
        cur += e.second;
        ans = max(ans, cur);
    }
    cout << ans << endl;
    return 0;
}
```

#### 经典题型
1. **最大重叠区间数**（如：会议室最少数量，最多同时在线人数）
2. **给定长度窗口，最多覆盖多少个区间**（本题）
3. **区间覆盖点数最少**（贪心，不用扫描线，但要对比记忆）
4. **矩形面积并**（扫描线 + 线段树 + 离散化）

#### 注意事项
- **区间类型**：如果题目给的是开区间、半开半闭，要调整事件位置（例如 `(B, -1)` 或 `(B+ε, -1)`）。
- **坐标类型**：坐标可能很大，要用 `long long` 存储。
- **排序细节**：如果出现坐标相同的事件，建议让 `+1`（开始）在 `-1`（结束）之前，这样 `cur` 在变化点能先加上再减去，避免答案漏掉峰值。
- **整数离散化**：有时需要把坐标离散化压缩，但简单扫描线可以直接用 `vector` 存事件排序，无需额外离散化。
- **不一定是闭区间**：如果区间覆盖定义是“点被包含”，就用闭区间处理；若定义为“边界不算”，则需调整端点坐标。

#### 如何培养扫描线思维
- 遇到“最多”“最少”和“区间”组合的问题时，先尝试把选择的对象（比如你选的区间的左端点）设成变量，写出每个给定区间对该变量的约束不等式，往往就会转化成区间覆盖问题。
- 画图！把区间画在数轴上，想象一条竖线从左扫到右，看哪些区间处于“活跃”状态。

#### 口诀
> **区间转参数，端点生事件。排序扫一遍，加加减减取最大。**

这份笔记记住，以后看到类似题目马上就能反应出“扫！”。有问题随时再问。

## 数学

### 乘法逆元

**乘法逆元**，简单说就是在模运算中，一个数的“倒数”。

在普通的算术里，一个数 \( a \) 的倒数是 \(\frac{1}{a}\)，因为 \( a \times \frac{1}{a} = 1\)。  
但在模运算的世界里，我们只关心余数，不能直接用小数。于是，就定义了一个整数叫**乘法逆元**，来充当“倒数”的角色。

---

#### 定义

对于整数 \(a\) 和正整数 \(m\)，如果存在整数 \(x\)，满足：

\[
a \cdot x \equiv 1 \pmod{m}
\]

那么 \(x\) 就叫 **\(a\) 模 \(m\) 下的乘法逆元**，常记作 \(a^{-1} \pmod{m}\) 或 \(\text{inv}(a)\)。

> 它是模意义下的倒数，因为乘起来同余 1。

---

#### 存在条件

**并不是所有数都有逆元**。  
\(a\) 有模 \(m\) 下的逆元，**当且仅当 \(a\) 和 \(m\) 互质**，即 \(\gcd(a, m) = 1\)。

- 比如 \(a=2,m=4\)，因为 \(2\) 和 \(4\) 不互质，就没有逆元。
- 如果 \(m\) 是质数（常见的情况，如 \(10^9+7\)），那么只要 \(a\) 不是 \(m\) 的倍数，就一定有逆元。

---

#### 为什么要用它？—— 代替除法

在模运算中，我们经常会遇到 \( (b / a) \bmod m \) 这样的计算。  
但取模运算里**不能直接做除法**，因为除法结果不一定是整数，而且取模分布律里除法不成立。

解决办法就是：**除以 \(a\) 等价于乘上它的逆元**。

\[
\frac{b}{a} \bmod m \;=\; \bigl(b \cdot a^{-1}\bigr) \bmod m
\]

这在组合数取模、数论等地方超级常用。比如求组合数 \(C_n^m \bmod p\)，就会把除法阶乘变成乘逆元。

---

#### 怎么求逆元？

常用三种方法：

##### (1) 扩展欧几里得算法（exgcd）—— 通用法
解同余方程 \(a x \equiv 1 \pmod{m}\) 等价于求 \(a x + m y = 1\) 的整数解 \(x\)。只要 \(\gcd(a,m)=1\)，exgcd 就能求出 \(x\)，把 \(x\) 调整到 \(0 \sim m-1\) 即得逆元。 
**适用于任意 \(m\)，不要求 \(m\) 是质数。**

##### (2) 费马小定理 —— 要求 \(m\) 是质数
若 \(m\) 是质数且 \(a \not\equiv 0 \pmod{m}\)，则有：

\[
a^{m-1} \equiv 1 \pmod{m}
\]

因此：
\[
a^{-1} \equiv a^{m-2} \pmod{m}
\]
用快速幂求 \(a^{m-2} \bmod m\) 就行。

前提条件

通常题目里会有一个全局的阶乘数组：
```cpp
f[0] = 1;
for (int i = 1; i <= n; i++)
    f[i] = f[i-1] * i % MOD;
```
即 `f[i] = i! mod MOD`。

现在是要求出每个 `f[i]` 对应的逆元，存到 `invf[i]` 里，使得 
`f[i] * invf[i] % MOD == 1`。

**重点**：MOD 必须是质数（比如 1e9+7），否则不能用费马小定理直接求逆。

---

第一步：快速幂求出最后一个逆元

```cpp
invf[n] = ksm(f[n], MOD - 2, MOD);
```
- 因为 MOD 是质数，且 `f[n]` 不是 MOD 的倍数（一般 n < MOD），由**费马小定理**：
  \[
  f[n]^{MOD-2} \bmod MOD
  \]
  就是 `f[n]` 的乘法逆元。
- 这就是 `invf[n] = (n!)^{-1} mod MOD`。

这一步是整个递推的起点。

---

第二步：倒着推出所有阶乘逆元

循环部分：
```cpp
for (int i = n - 1; i >= 0; i--) {
    invf[i] = invf[i + 1] * (i + 1) % MOD;
}
```

推导过程（核心就是这条恒等式）：
\[
(i!)^{-1} \equiv ((i+1)!)^{-1} \cdot (i+1) \pmod{MOD}
\]

**为什么成立？**
- 因为 `(i+1)! = i! * (i+1)`
- 两边同时取逆元：
  \[
  ((i+1)!)^{-1} \equiv (i! \cdot (i+1))^{-1} \equiv (i!)^{-1} \cdot (i+1)^{-1}
  \]
- 这等价于：
  \[
  (i!)^{-1} \equiv ((i+1)!)^{-1} \cdot (i+1)
  \]
- 写成代码就是 `invf[i] = invf[i+1] * (i+1) % MOD`。

**从乘法角度理解更简单**：
`invf[i]` 是 `1/(i!)`，`invf[i+1]` 是 `1/((i+1)!)`。 
`1/(i!)` 比 `1/((i+1)!)` 多乘一个 `(i+1)`，所以直接用 `invf[i+1] * (i+1)` 就能得到 `invf[i]`。

---

为什么这样写效率高？

- 如果每个 `f[i]` 都单独用一次快速幂求逆，总复杂度是 \(O(n \log MOD)\)。
- 现在只用一次快速幂 \(O(\log MOD)\)，再加上 \(O(n)\) 的倒序循环，就到了线性复杂度。

这和你之前问的**线性递推求逆元**的思想其实一脉相承，都是利用“大数的逆元可以由小数的逆元推出来”，只不过这里是在阶乘上做递推，而不是直接对 `1..n` 做递推。

简单总结： 
`invf[n]` 用费马小定理求出 → 利用 `1/i! = 1/(i+1)! * (i+1)` 这个恒等式，从后向前一次乘法就得到所有阶乘逆元。

##### (3) 线性递推 —— 求 \(1 \sim n\) 所有逆元
当 \(m\) 为质数时，可以 O(n) 求出 \(1,2,\dots,n\) 的逆元：

\[
\text{inv}[i] = \left( m - \left\lfloor \frac{m}{i} \right\rfloor \right) \times \text{inv}[m \bmod i] \bmod m
\]

常用于预处理阶乘逆元等。

#### 证明

##### 费马小定理

费马小定理是数论和算法竞赛里的基石，它的证明方法有好几种。这里介绍最经典、也最直观的一种：**基于“缩剩余系”的排列法**，它不需要太多预备知识。

---

定理回顾

若 \(p\) 是质数，且整数 \(a\) 不被 \(p\) 整除，则：
\[
a^{p-1} \equiv 1 \pmod{p}
\]
等价形式：对任意整数 \(a\)，有 \(a^p \equiv a \pmod{p}\)。

---

证明：排列法（缩系相乘）

1. 构造集合 S

考虑模 \(p\) 下所有非零余数构成的集合：
\[
S = \{1, 2, 3, \dots, p-1\}
\]
它的大小是 \(p-1\)，且其中每个数都与质数 \(p\) 互质。

2. 用 a 去乘集合中的每个元素

把 \(S\) 中的每个数都乘上 \(a\)，得到新集合：
\[
T = \{a \cdot 1 \bmod p, \; a \cdot 2 \bmod p, \; \dots, \; a \cdot (p-1) \bmod p\}
\]

3. 证明 T 就是 S 的一个排列

验证三点：
- **不会出现 0**：因为 \(a\) 和 \(S\) 中的数都与 \(p\) 互质，乘积不可能被 \(p\) 整除，所以模 \(p\) 后不可能为 0。
- **元素都在 1~p-1 之间**：模 \(p\) 后余数自然落在这个范围。
- **两两不同（单射）**：假设 \(a \cdot x \equiv a \cdot y \pmod{p}\)，由于 \(\gcd(a,p)=1\)，乘法消去律成立，两边约掉 \(a\) 得到 \(x \equiv y \pmod{p}\)。既然 \(x,y\) 原本就在 \(1 \sim p-1\) 中，它们必然相等。

因此 \(T\) 是 \(S\) 中元素的**重排列**，\(T\) 和 \(S\) 作为集合完全一样，只是顺序不同。

4. 乘积相等

既然两个集合相同，它们的元素乘积必然同余：
\[
\prod_{x \in S} x \equiv \prod_{x \in S} (a \cdot x) \pmod{p}
\]
左边就是 \((p-1)!\)。右边提取 \(a\)：
\[
\prod_{x=1}^{p-1} (a \cdot x) \equiv a^{p-1} \cdot \prod_{x=1}^{p-1} x \equiv a^{p-1} \cdot (p-1)! \pmod{p}
\]
于是得到：
\[
(p-1)! \equiv a^{p-1} \cdot (p-1)! \pmod{p}
\]

5. 消去 (p-1)!

因为 \(p\) 是质数，\((p-1)!\) 的每个因子 \(1,2,\dots,p-1\) 都小于 \(p\) 且不被 \(p\) 整除，所以 \((p-1)!\) 和 \(p\) 互质。在模 \(p\) 下，互质的数可以消去（乘以它的逆元即可）。两边约掉 \((p-1)!\)：
\[
1 \equiv a^{p-1} \pmod{p}
\]
证明完毕。

---

另一种常见证法：二项式定理 + 归纳法

简单提一下思路，也很好懂：  
要用数学归纳法证 \(a^p \equiv a \pmod{p}\)。  
基础 \(a=0\) 或 \(1\) 显然。  
假设对 \(a\) 成立，看 \(a+1\)：
\[
(a+1)^p = a^p + \binom{p}{1}a^{p-1} + \cdots + \binom{p}{p-1}a + 1
\]
所有二项式系数 \(\binom{p}{k} (1 \le k \le p-1)\) 都含因子 \(p\)（因为 \(p\) 是质数），所以这些中间项模 \(p\) 全为 0。  
于是 \((a+1)^p \equiv a^p + 1 \pmod{p}\)。  
再由归纳假设 \(a^p \equiv a\)，得 \((a+1)^p \equiv a+1 \pmod{p}\)。  
归纳成立，故对所有非负整数 \(a\) 有 \(a^p \equiv a\)。若 \(p \nmid a\)，两边消去 \(a\) 即得 \(a^{p-1} \equiv 1\)。

---

从定理到乘法逆元

既然 \(a^{p-1} \equiv 1 \pmod{p}\)，那么
\[
a \cdot a^{p-2} \equiv 1 \pmod{p}
\]
因此 **\(a^{p-2} \bmod p\) 就是 \(a\) 在模质数 \(p\) 下的乘法逆元**，可以用快速幂在 \(O(\log p)\) 内求出。

下面来详细拆解这两种方法的证明，它们都非常巧妙。

---

##### 裴蜀定理

**贝祖定理**（也叫贝祖恒等式、裴蜀定理）是数论里的一个基本结论，它保证了**两个整数的最大公约数可以用这两个数的整数倍之和表示**。 
正是因为有它，扩展欧几里得算法（exgcd）才能用来求乘法逆元。

---

定理内容

对于任意非零整数 \(a\) 和 \(b\)，一定存在整数 \(x\) 和 \(y\)，使得：

\[
a \cdot x + b \cdot y = \gcd(a, b)
\]

并且，**\(\gcd(a,b)\) 是 \(a\) 和 \(b\) 的线性组合能表示出的最小正整数**。

---

证明思路（用 exgcd 的过程）

这个定理的证明就是扩展欧几里得算法的构造过程：

1. 辗转相除： 
   \(a\) 除 \(b\) 得商和余数：\(a = b \cdot q + r\)，且 \(\gcd(a, b) = \gcd(b, r)\)。
2. 反复迭代，直到余数为 0，此时最大公约数就是当前的除数（比如 \(b'\)）。
3. 回溯代入： 
   从最后一对 \((b', 0)\) 的显然解 \(b' \cdot 1 + 0 \cdot 0 = b'\) 开始，逐步代回去，就能构造出原始 \(a, b\) 的系数 \(x, y\)。

这个过程就是前面我们讲的 exgcd 流程，它**同时证明了存在性并给出了构造**。

---

与逆元的直接关系

求 \(a\) 模 \(m\) 的乘法逆元，本质是解：  
\[
a \cdot x \equiv 1 \pmod{m}
\]
写成扩展形式就是：存在整数 \(y\)，使得  
\[
a \cdot x + m \cdot y = 1
\]

只有当 \(\gcd(a, m) = 1\) 时，根据贝祖定理，这样的 \(x, y\) 才一定存在。此时 \(x \bmod m\) 就是所求的逆元。 
如果 \(\gcd(a, m) \neq 1\)，右侧必须是 \(\gcd\) 的倍数才可能有解，而逆元要求等于 1，说明**不互质时一定没有逆元**。

这也解释了前面反复强调的“逆元存在的前提是 \(a\) 与 \(m\) 互质”。

---

推广：裴蜀定理的一般形式

- **多个整数**： 
  对于整数 \(a_1, a_2, \dots, a_n\)，也存在 \(x_1, \dots, x_n\) 使得  
  \[
  a_1 x_1 + a_2 x_2 + \dots + a_n x_n = \gcd(a_1, a_2, \dots, a_n)
  \]
- **可解性判断**： 
  不定方程 \(a x + b y = c\) 有整数解的充要条件是 \(\gcd(a, b) \mid c\)。

---

记一个简单例子

\(a=14, b=10\)，\(\gcd(14,10)=2\)。 
能找到：\(14 \times (-2) + 10 \times 3 = -28+30 = 2\)，系数 \(x=-2, y=3\)。  

回到逆元的场景，若模数 \(m=7\)，\(a=3\)，\(\gcd(3,7)=1\)： 
需要 \(3x + 7y = 1\)，exgcd 可得 \(3 \times (-2) + 7 \times 1 = 1\)，所以 \(-2 \equiv 5 \pmod{7}\) 即逆元。  

**总结**：贝祖定理保证了互质时逆元必然存在，exgcd 则把它算出来。

##### 扩展欧几里得算法（exgcd）

**目标**：已知 \(\gcd(a, m) = 1\)，求一个 \(x\) 使得 \(a \cdot x \equiv 1 \pmod{m}\)。

1. 转化为不定方程

同余式等价于存在整数 \(y\)，满足：
\[
a \cdot x + m \cdot y = 1
\]
因为 \(\gcd(a, m) = 1\)，根据**贝祖定理**，这样的整数解 \((x, y)\) 一定存在。我们只要找到它，\(x \bmod m\) 就是逆元。

2. 算法如何构造解 —— 归纳/递归证明

扩展欧几里得算法基于带余除法，递归地求解。其核心是： 
如果我们要解 \(a x + m y = \gcd(a, m)\)，已知 \(\gcd(a, m) = \gcd(m, a \bmod m)\)，可以先求较小的那一组解。

递归函数 `exgcd(a, b)` 返回 \((g, x, y)\) 满足 \(a x + b y = g = \gcd(a,b)\)：

- **边界**：当 \(b = 0\) 时，\(\gcd(a, 0) = a\)，此时方程 \(a \cdot 1 + 0 \cdot 0 = a\)，所以一组解是 \(x=1, y=0\)。
- **递归**：对 \(b \ne 0\)，先递归调用 `exgcd(b, a % b)`，得到解 \(x', y'\)，满足：
  \[
  b \cdot x' + (a \bmod b) \cdot y' = \gcd(b, a \bmod b) = \gcd(a, b)
  \]
  利用 \(a \bmod b = a - \lfloor a/b \rfloor \cdot b\)，代入：
  \[
  b x' + \left(a - \lfloor a/b \rfloor \cdot b\right) y' = g
  \]
  展开并重新结合 \(a\) 和 \(b\)：
  \[
  a \cdot y' + b \cdot \left( x' - \lfloor a/b \rfloor \cdot y' \right) = g
  \]
  对比标准形式 \(a x + b y = g\)，我们得到了 \(a,b\) 对应的解：
  \[
  x = y', \qquad y = x' - \lfloor a/b \rfloor \cdot y'
  \]
  这就完成了“递归回溯构造解”的证明。

3. 得到逆元

因为我们求的是 \(a \cdot x + m \cdot y = 1\)，令 `exgcd(a, m)` 返回的 \(x\) 即满足该等式。  
把 \(x\) 调整到 \(0 \sim m-1\) 的范围（比如 `(x % m + m) % m`），就得到了 \(a\) 模 \(m\) 的乘法逆元。

---

##### 线性递推

**前提**：\(p\) 是一个质数，我们要快速求 \(1,2,\dots,n\) 每个数在模 \(p\) 下的逆元（通常 \(n < p\)）。

**递推公式**：
\[
\text{inv}[i] = \left( p - \left\lfloor \frac{p}{i} \right\rfloor \right) \cdot \text{inv}[p \bmod i] \bmod p
\]
边界 \(\text{inv}[1] = 1\)。

证明推导

1. 对于任意 \(i \in [2, n]\)，用 \(i\) 除 \(p\) 做带余除法：
   \[
   p = k \cdot i + r, \quad \text{其中 } k = \left\lfloor \frac{p}{i} \right\rfloor,\; r = p \bmod i
   \]
   显然 \(0 < r < i\)，因为 \(p\) 是质数，\(i\) 不整除 \(p\)。

2. 将等式放在模 \(p\) 下观察：
   \[
   k \cdot i + r \equiv 0 \pmod{p}
   \]
   即：
   \[
   k \cdot i \equiv -r \pmod{p}
   \]

3. 为了解出 \(i^{-1}\)，两边同时乘以 \(i^{-1}\) 和 \(r^{-1}\)（因为 \(p\) 是质数，且 \(i, r\) 均与 \(p\) 互质，逆元都存在）：
   \[
   k \cdot r^{-1} \equiv -i^{-1} \pmod{p}
   \]
   整理得到：
   \[
   i^{-1} \equiv -k \cdot r^{-1} \pmod{p}
   \]

4. 将 \(k = \lfloor p/i \rfloor\)，\(r = p \bmod i\) 代回：
   \[
   i^{-1} \equiv -\left\lfloor \frac{p}{i} \right\rfloor \cdot (p \bmod i)^{-1} \pmod{p}
   \]

5. 为了让结果落在非负范围，用 \(p\) 减去负数部分：
   \[
   i^{-1} \equiv \left( p - \left\lfloor \frac{p}{i} \right\rfloor \right) \cdot (p \bmod i)^{-1} \pmod{p}
   \]
   注意到 \(r = p \bmod i < i\)，因此 \((p \bmod i)\) 的逆元在之前已经求出，可以递推。  
   边界 \(i=1\) 时，\(1^{-1} \equiv 1 \pmod{p}\)。

这就严格证明了线性递推公式的正确性。

---

小结

- **exgcd** 利用辗转相除和贝祖定理，通过回溯系数构造出解，适用于任何 \(\gcd(a,m)=1\) 的模数。
- **线性递推** 利用带余除法恒等式 \(p = k i + r\)，巧妙地用更小的逆元表示当前逆元，在 \(O(n)\) 内得到连续逆元，要求模数为质数。

#### 举例

模 \(7\)：
- \(3 \times 5 = 15 \equiv 1 \pmod{7}\)，所以 \(3\) 的逆元是 \(5\)。
- 如果要算 \( \frac{2}{3} \bmod 7 \)，就算 \(2 \times 5 = 10 \equiv 3 \pmod{7}\)，这就是结果。

模 \(10^9+7\)（质数）：
- 求 \(a\) 的逆元直接用 `pow(a, mod-2, mod)` 快速幂即可。

---

#### 总结

- **乘法逆元**：模 \(m\) 下满足 \(a \cdot x \equiv 1 \pmod{m}\) 的整数 \(x\)，记作 \(a^{-1}\)。
- **存在前提**：\(\gcd(a,m)=1\)。
- **核心用途**：把模除法变成乘法，解决组合数、线性同余等各类问题。
- **常用求法**：exgcd（通用）、费马小定理（m为质数）、线性递推（批量求逆元）。

### 质因数分解

#### 质因数分解
- 任意正整数 \(n > 1\) 可唯一拆成 \(n = p_1^{e_1} p_2^{e_2} \dots p_k^{e_k}\)。
- 代码：
```cpp
map<long long, int> mp; // 质数 -> 指数
void factor(long long n) {
    for (long long i = 2; i * i <= n; i++) {
        if (n % i == 0) {
            int cnt = 0;
            while (n % i == 0) n /= i, cnt++;
            mp[i] += cnt;
        }
    }
    if (n > 1) mp[n] += 1;
}
```
- 复杂度 \(O(\sqrt{n})\)。

#### 因子数量 & 因子和
设 \(n = \prod p_i^{e_i}\)：
- 因子数量：\(\prod (e_i+1)\)
- 因子和：\(\prod \frac{p_i^{e_i+1}-1}{p_i-1}\)
- 可用等比数列求和理解。

#### 生成所有因子
**核心思想**：每个因子由每个质数选 0 到 \(e_i\) 次方组合而成。

##### 递归 (DFS)
```cpp
void dfs(int idx, long long cur) {
    if (idx == primes.size()) {
        divisors.push_back(cur);   // 得到一个完整因子
        return;
    }
    long long p = primes[idx].first;
    int e = primes[idx].second;
    long long mult = 1;
    for (int i = 0; i <= e; i++) {
        dfs(idx + 1, cur * mult);
        mult *= p;   // p的1,2,...次方
    }
}
```

##### 迭代 (二重循环或队列)
```cpp
vector<long long> divs = {1};
for (auto &[p, e] : primes) {
    int sz = divs.size();
    long long mult = 1;
    for (int i = 1; i <= e; i++) {
        mult *= p;
        for (int j = 0; j < sz; j++) {
            divs.push_back(divs[j] * mult);
        }
    }
}
```
- 两种方法都能不重不漏地生成所有因子。

#### 模运算下的因子处理技巧
问题：当因子 \(d\) 巨大（比如 \(10^{100}\)）时，如何求 \(d^d \pmod{mod}\)？
- 底数取模：\(d \bmod mod\)
- 指数降幂：当 \(mod\) 为质数且 \(\gcd(d, mod)=1\) 时，用费马小定理 \(d^{mod-1} \equiv 1\)，指数可对 \(mod-1\) 取模。
- 实现时在 DFS 中同时维护 `cur % mod` 和 `cur % (mod-1)`。
- 特判：若 `cur % mod == 0`，该因子贡献为 0。

#### 合并两个数的因子分解
- 若要求 \(x \times y\) 的因子，分别分解 \(x, y\)，把同一质数的指数相加即可。无需把乘积分出来再分解（乘积可能过大）。

#### 常见题型及应对
- **求所有因子的和/幂和/乘积**：生成因子时直接累加运算（必要时取模）。
- **求恰好有 k 个因子的数**：反向构造，枚举指数组合。
- **区间内因子数最多的数**（反质数）：DFS 搜索质数指数组合，利用因子数公式剪枝。
- **模数巨大时的因子相关求和**：必须边生成因子边取模，避免存储完整大数。

#### 复杂度与优化
- 质因数分解 \(O(\sqrt{n})\)。
- 因子生成 \(O(d(n))\)，其中 \(d(n)\) 为因子个数，一般 \(d(n) \ll n^{1/3}\)，最坏情况（高复合数）约 \(10^5 \sim 10^6\) 级别。
- 若因子数量过大（如 \(10^7\)），需要采用杜教筛等数论方法，但一般竞赛中 DFS 足够。

---

**核心口诀**： 
分解质因数拿指数，DFS 组合出全部因子，大指数用费马降幂，模运算一路跟随。

### 组合数学

#### Dilworth 定理

##### 基本概念

**偏序集**：集合 \(P\) 上定义了一个二元关系 \(\le\)，满足：
- 自反性：\(a \le a\)
- 反对称性：若 \(a \le b\) 且 \(b \le a\)，则 \(a = b\)
- 传递性：若 \(a \le b\) 且 \(b \le c\)，则 \(a \le c\)

不是任意两个元素都能比较，能比则称**可比**，不能则称**不可比**。

**链**：一个子集，其中任意两个元素都可比（能排成全序）。
**反链**：一个子集，其中任意两个**不同**元素都不可比。

**链覆盖**：将整个集合划分成若干条链（每个元素恰好属于一条链）或覆盖（至少属于一条链）。 
**最小链覆盖数**：所需的最少链的条数。

**最大反链大小**：所有反链中包含元素个数的最大值。

---

#####  定理内容

> **Dilworth 定理**（有限偏序集版本）： 
> 在任意**有限**偏序集中，最小链覆盖数等于最大反链的大小。

用符号表示：  
\[
\text{最小链覆盖数} = \text{最大反链的元素个数}
\]

---

##### 直观理解

- **“容易的一半”**：一条链里最多只能包含一个反链中的元素（因为反链元素两两不可比，不能在同一个全序中）。 
  \(\Rightarrow\) 任何链覆盖的链数 \(\ge\) 任何反链的大小。 
  \(\Rightarrow\) 最小链覆盖数 \(\ge\) 最大反链大小。

- **“困难的一半”**：我们总是可以只用恰好等于最大反链大小的链数来覆盖整个集合。 
  这意味着最大反链给出了所需链数的**精确下界**，并且这个下界是可以达到的。

---

##### 证明思路（归纳构造法）

对集合大小 \(n\) 归纳。

- \(n=1\) 显然成立。
- 假设对少于 \(n\) 个元素的偏序集成立，现考虑 \(n\) 个元素的偏序集 \(P\)。

1. 取一个**极大元** \(m\)（没有元素比它大）。
2. 考虑 \(P' = P \setminus \{m\}\)，由归纳假设，\(P'\) 的最小链覆盖数 \(k\) 等于其最大反链大小。
3. 设 \(P\) 的最大反链大小为 \(A\)。有两种情况：
   - **情况1**：\(A = k\)。 
     说明添加 \(m\) 后没有形成更大的反链。此时可以证明，\(P'\) 中所有 \(\le m\) 的元素都在同一条链里，于是把 \(m\) 接到该链末尾，仍用 \(k\) 条链覆盖 \(P\)，且 \(A = k\)，等式成立。
   - **情况2**：\(A = k+1\)。 
     说明存在包含 \(m\) 的大小为 \(k+1\) 的反链。此时我们新开一条链只放 \(m\)，其余用 \(k\) 条链覆盖，总共 \(k+1\) 条链，等于新的最大反链大小。

这就完成了归纳。

---

##### 对偶形式：Mirsky 定理

与 Dilworth 定理“对偶”的定理是 **Mirsky 定理**：
> 在有限偏序集中，最小反链覆盖数 = 最大链的大小。

即：用最少的反链覆盖所有元素，所需反链数等于最长链的长度。在有些问题中也会用到。

---

##### 算法竞赛中的常见应用

**场景**：当一个问题的约束可以转化为二维偏序 \(a_i \le a_j\) 且 \(b_i \le b_j\)，要求最少需要多少条“单调不减路径”覆盖所有元素。

**标准解法**：
1. 将所有元素按第一维（如 \(V\)）升序排序（第一维相同按第二维升序）。
2. 此时第一维自动满足不降，问题简化为在第二维序列上求**最小链覆盖**。
3. 由 Dilworth 定理，最小链覆盖数 = 第二维序列的**最长严格下降子序列 (LDS) 长度**。
4. LDS 可通过将第二维取负后求最长上升子序列 (LIS) 在 \(O(N\log N)\) 内解决。

---

##### 注意事项

- Dilworth 定理要求**有限**偏序集，竞赛中的 \(N\) 总是有限的，所以可以直接用。
- 排序时若第一维相同，必须按第二维升序，否则反链会被错误地拉长（相同第一维但第二维不同的元素原本是可比还是不可比需结合定义判断，一般按照偏序定义处理）。
- 最终求的是**严格下降**还是**不增**取决于反链的定义：若偏序允许相等时可比，则反链要求严格不满足偏序，即严格下降。

---

#####  一句话记忆

**“最少链覆盖 = 最长反链，只需按一维排序，另一维求 LDS。”**
