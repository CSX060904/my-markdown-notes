#### 牛客小红的网格路径 II

`[F-小红的网格路径 II_牛客周赛 Round 143](https://ac.nowcoder.com/acm/contest/134529/F)`

题目特征识别

| 特征     | 本题情况                                             |
| -------- | ---------------------------------------------------- |
| 网格大小 | n, m 可达 1e9（极大）                                |
| 移动限制 | 向上、向下、向右，**不能向左**，**不能重复访问格子** |
| 障碍     | 每列至多一个障碍（极稀疏），k ≤ 1e5                  |
| 目标     | 从 (1,1) 到 (n,m) 的路径方案数 mod 1e9+7             |

**核心矛盾**：允许向上/向下导致组合公式失效；但“不能向左 + 不重复”又强制每列经过的格子是一段**连续区间**，这提供了状态压缩的可能。

---

为什么不能用组合计数？

- 组合计数通常要求每一步是**独立的单调移动**（如只有右、下），这样路径数 = 排列组合。
- 本题允许上下移动，同一列内可以走一段竖线，**不同列的竖线区间存在衔接关系**（上一列的离开行 = 下一列的进入行），无法拆成独立的列相乘。
- 这种“列间依赖”本质是一个**顺序决策过程**，必须用 DP 按列递推。

---

DP 状态设计核心

**1. 状态定义**
- `dp[i][j]`：到达第 i 列、行 j 的方案数（作为该列的“进入行”）。
- 递推：从第 i 列的状态算出第 i+1 列所有行的方案数。

**2. 状态压缩的关键观察**
- 一列中如果没有障碍，该列所有行的 dp 值**完全相同**（都等于上一列的总和）。
- 如果有一列有一个障碍在 x 行，则该列被 x 分割成上方和下方两部分，**各自内部所有行的 dp 值相等**，记作：
  - `up`：障碍上方每一行的方案数
  - `down`：障碍下方每一行的方案数
- 因此，整个列的状态只需两个变量 `(up, down)`。

**3. 空列的快速幂跳过**
- 连续 d 列空列：每列会将上一列的总和复制到本列所有行，所以经过 d 列后，每一行的值 = (最初总和) × n^(d-1)。
- 这样 O(d) 的转移被压缩成 O(log d)。

---

转移方程详解

设当前障碍列 `(x, y)`，其状态为 `u`（上）、`d`（下）。下一个障碍在 `(nx, ny)`。

情况 A：中间有空列（ny - y > 1）

1. 当前列总和：`sum = (x-1)*u + (n-x)*d`
2. 空列数：`gap = ny - y - 1`
3. 进入 ny 列前一列时，所有行的值为：`v = sum * n^{gap-1}`
4. 在新障碍列按行数拆分：
   - `new_u = (nx - 1) * v`
   - `new_d = (n - nx) * v`

情况 B：相邻障碍列（ny == y + 1）

根据新旧障碍的行位置分三种：
- **nx == x**（同一行）  
  `new_u = (x-1)*u`  
  `new_d = (n-x)*d`
- **nx > x**（新障碍在下方）  
  `new_d = (n-nx)*d`  
  `new_u = (x-1)*u + (nx-x-1)*d`
- **nx < x**（新障碍在上方）  
  `new_u = (nx-1)*u`  
  `new_d = (n-x)*d + (x-nx-1)*u`

---

代码与算法步骤一一对应

```cpp
// 1. 快速幂
ll ksm(ll a, ll b) { ... }

// 2. 读入障碍，按列排序
vector<pair<ll,ll>> a;  // {y, x}
sort(a.begin(), a.end());

// 3. 初始化第一个障碍列
auto [y0, x0] = a[0];
if (y0 == 1) { u = 1; d = 0; }
else {
    ll v = ksm(n, y0 - 2);
    u = (x0 - 1) * v % mod;
    d = (n - x0) * v % mod;
}
ll px = x0, py = y0;

// 4. 转移函数（对应上述情况）
auto step = [&](ll x, ll y, ll u, ll d, ll nx, ll ny) -> pair<ll,ll> {
    if (ny - y > 1) {
        ll sum = (u*(x-1) + d*(n-x)) % mod;
        ll v = sum * ksm(n, ny-y-2) % mod;
        return {(nx-1)*v % mod, (n-nx)*v % mod};
    } else {
        if (nx == x) return {(nx-1)*u % mod, (n-nx)*d % mod};
        else if (nx > x) {
            ll new_d = (n-nx)*d % mod;
            ll new_u = ((x-1)*u + (nx-x-1)*d) % mod;
            return {new_u, new_d};
        } else {
            ll new_u = (nx-1)*u % mod;
            ll new_d = ((n-x)*d + (x-nx-1)*u) % mod;
            return {new_u, new_d};
        }
    }
};

// 5. 依次处理障碍
for (int i = 1; i < k; i++) {
    auto [ny, nx] = a[i];
    tie(u, d) = step(px, py, u, d, nx, ny);
    px = nx; py = ny;
}

// 6. 到达终点列
if (py != m) {
    ll sum = (u*(px-1) + d*(n-px)) % mod;
    u = d = sum * ksm(n, m-py-1) % mod;
}
cout << d << endl;   // d = 到达终点行 n 的方案数
```

---

同类题型通用解题框架

**触发条件**（看到以下特征立刻想到这个方法）：
1. 网格路径，移动规则有“不能向左”且“不能重复访问格子”。
2. 网格行列极大，但障碍（或特殊列）数量很少。
3. 每列最多一个障碍。

**解题四步曲**：
1. **分析列内路径形状**：必为连续区间。
2. **定义列状态**：障碍上、下方的单位方案数（因为同侧各行的值相同）。
3. **设计相邻列转移**：分有空列和无空列两种，用快速幂跳过空列。
4. **边界处理**：第一列和最后一列单独考虑。

**延伸思考**：
- 若每列可能有多个障碍？则区间会被切得更碎，状态数增加，但仍可用线段树维护每个连续段的 dp 值。
- 若移动规则只有右和下（不允许向上）？退化为经典组合数问题，障碍则用容斥或 DP。

---

个人心得记录

- **组合 vs DP**：一旦看到“可向上/向下”且“不能重复”，立刻放弃组合公式，转向按列 DP。
- **压缩状态**：多画图观察同一列中哪些行的 dp 值相同，往往能大幅减少状态数。
- **快速幂**：极大行列时，连续相同状态的阶段一定要用快速幂跳过，否则复杂度爆炸。
- **取模细节**：每步加法、乘法后都要 `% mod`，避免 `long long` 溢出。

**像这种网格计数问题，要么用暴力(搜索，递归)直接来做，要么用dp(通常是状压DP，线性DP来做)，要么用计数原理来做(符合独立性原则的时候)。**

---

**AC代码：**

```c++
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
const ll mod = 1e9+7;

ll ksm(ll a,ll b){
    a %= mod;
    ll p = 1;
    while(b != 0){
        if(b&1){
            p = p*a%mod;
        }
        a = a*a%mod;
        b = b>>1;
    }
    return p;
}

void solve() {
    ll n,m,k;
    cin >> n >> m >> k;
    if(k == 0){
        cout << ksm(n,m-1) << endl;
        return;
    }
    vector<pair<ll,ll>>a;
    ll x,y;
    for(int i = 1;i <= k;i++){
        cin >> x >> y;
        a.push_back({y,x});
    }
    sort(a.begin(),a.end());
    ll u,d;
    auto p = a[0];
    y = p.first,x = p.second;
    if(y == 1){
        u = 1,d = 0;
    }
    else{
        ll sum = ksm(n,y-2);
        u = (x-1)*sum%mod;
        d = (n-x)*sum%mod;
    }
    auto step = [&](ll x,ll y,ll u,ll d,ll nx,ll ny)->pair<ll,ll>{
        if(ny-y > 1){
            ll sum = (u*(x-1)%mod+d*(n-x)%mod)%mod;
            u = d = sum*ksm(n,ny-y-2)%mod;
            u = (nx-1)*u%mod;
            d = (n-nx)*d%mod;
            return {u,d};
        }
        else{
            if(nx == x){
                u = (nx-1)*u%mod;
                d = (n-nx)*d%mod;
            }
            else if(nx > x){
                ll t = d;
                d = (n-nx)*d%mod;
                u = (x-1)*u%mod+(nx-x-1)*t%mod;
            }
            else{
                ll t = u;
                u = (nx-1)*u%mod;
                d = (n-x)*d%mod+(x-nx-1)*t%mod;
            }
            return {u,d};
        }
    };
    ll px = x,py = y;
    for(int i = 1;i < a.size();i++){
        y = a[i].first;
        x = a[i].second;
        auto r = step(px,py,u,d,x,y);
        u = r.first;
        d = r.second;
        px = x,py = y;
    }
    if(py != m){
        ll sum = (u*(x-1)%mod+d*(n-x)%mod)%mod;
        u = d = sum*ksm(n,m-py-1)%mod;
    }
    cout << d << endl;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr); 
    solve();
    return 0;
}

```

### 树形DP

#### 牛客小苯的DFS

`[小苯的DFS](https://ac.nowcoder.com/acm/problem/317234)`

题目本质是：**在“父-子-兄弟”这个 DFS 顺序下，随机打乱兄弟顺序，求序列单调不降的概率**。

自底向上 DP 思路

对每个节点 `u`，我们想要两个信息：

- `p[u]`：以 `u` 为根的子树 DFS 序列单调不降的概率
- `mn[u]`, `mx[u]`：在**已经满足单调不降**的条件下，这个子树序列的最小值和最大值（其实就是整个子树的点权最小值和最大值）

**递推过程**：

1. 若 `u` 是叶子：`p = 1`, `mn = mx = a[u]`。
2. 若 `u` 有 `k` 个子节点：
   - 先递归计算所有子节点，只要有任一 `p[v] == 0`，直接 `p[u] = 0` 并返回。
   - 收集每个子树的 `[mn[v], mx[v]]`，并将这些区间按**左端点**排序。
   - 检查整体能否形成非降链：
     - `a[u] ≤ mn[第一个儿子]`
     - 对所有相邻儿子 `i`，满足 `mx[第i个儿子] ≤ mn[第i+1个儿子]`
   - 若违反任何一条，`p[u] = 0`。
   - 若满足，计算排列因子：完全相同区间可以互换。设每种相同区间出现次数为 cj*c**j*，则满足条件的排列数为 ∏(cj!)∏(*c**j*!)。总排列数为 k!*k*!。因此该层的概率因子为 ∏cj!k!*k*!∏*c**j*!。
   - `p[u] = (∏ p[v]) × 因子 mod 998244353`
   - `mn[u] = a[u]` （因为此时 `a[u]` 必为全局最小）
   - `mx[u] = mx[最后一个儿子]` （即排序后最后一个区间的右端点）

为什么这样是对的？

DFS 序是先根，然后按**某个随机排列**依次拼接各子树的 DFS 序。要让整个序列单调不降，必须同时满足：

- 根的值 ≤ 第一个子树的最小值
- 每个子树内部单调（由 `p[v]` 保证）
- 前一个子树的最大值 ≤ 后一个子树的最小值

子树内部一旦单调不降，其序列必然从小到大排列整个子树的值，因此最小/最大值就是子树点权的最值。于是子树可以抽象成一个值区间，兄弟之间就是这些区间的排列问题，概率计算变成一个纯粹的组合问题。

------

如何避免“漏判”和“想不到”——这类题的方法论

这类**树上随机 DFS 序 + 序列单调性**的题目，关键是**把全局单调性拆成局部约束**。你可以按以下步骤系统地分析：

1. 写出 DFS 序的结构

DFS 序 = `根 + 子树1全部 + 子树2全部 + ...`
单调不降 ⇒

- 根 ≤ 子树1首元素
- 子树1尾元素 ≤ 子树2首元素
- 子树2尾元素 ≤ 子树3首元素
- ……

2. 识别需要的“子树信息”

为了让父节点能判断上述条件，每个子树需要提供：

- 这个子树的**首元素**（最小值）
- 这个子树的**尾元素**（最大值）
- 这个子树内部合法的概率

只要这三个信息能递归计算，DP 就是可行的。

3. 把约束转化为组合条件

兄弟间的顺序是均匀随机的，问题变成：
**有 k 个带标记区间，求将它们排成非降链的排列数占比。**
常见技巧：

- 检查是否能排成链：按左端点排序，逐一检查 `R_i ≤ L_{i+1}`
- 计数合法排列数：完全相同的区间可以互换，因此是 `∏ cnt!`，除以 `k!` 得到概率因子。

4. 避免漏判的检查清单

每次写这种题，可以用一个**清单**逐项对照：

- 根与第一个儿子的最小值关系（`a[u] <= mn[first]`）
- 相邻儿子之间的“尾 ≤ 头”关系（`mx[i] <= mn[i+1]`）
- 是否考虑了子节点概率为 0 的短路情况
- 对于所有儿子都相同值的边界情况，概率因子是否正确（阶乘是否计算对）
- 叶子节点、单子节点、全树等特殊情况是否覆盖

5. 调试技巧

用**小样例画图模拟**，手算答案，然后与程序输出对比。打印中间 DP 值，看 `mn, mx, p` 是否合理。
尤其要构造几个容易出错的例子：

- 值全相等（应该输出 1）
- 只有两个儿子，值分别为 `[2,1]`（应输出 0）
- 三层树，兄弟区间有交错的情况

这类题的精髓就是**把全局排序转化为子树的区间拼接**，一旦你抽象出“区间链排列”这个组合模型，后续就是熟练度的问题了。多练习几道类似的树形概率/计数题（如随机排列兄弟求某种序的概率），这种思维方式会逐渐变成直觉。

**对于这种题，一般就是先判断可行性，再判断个数。**

**AC代码：**

```c++
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
const ll mod = 998244353;
ll f[200005];
ll invf[200005];

ll ksm(ll a,ll b){
    ll p = 1;
    while(b != 0){
        if(b&1)p = a*p%mod;
        a = a*a%mod;
        b = b>>1;
    }
    return p;
}

void ff(){
    ll p = 1;
    for(int i = 1;i <= 200000;i++){
        p = p*i%mod;
        f[i] = p;
    }
    invf[200000] = ksm(f[200000],mod-2);
    for(int i = 199999;i > 0;i--){
        invf[i] = invf[i+1]*(i+1)%mod;
    }
}

void solve() {
    int n;
    cin >> n;
    vector<int>a(n+1,0);
    for(int i = 1;i <= n;i++){
        cin >> a[i];
    }
    vector<vector<int>> g(n + 1);
    int x,y;
    for(int i = 1;i <= n-1;i++){
        cin >> x >> y;
        g[x].push_back(y);
        g[y].push_back(x);
    }
    vector<int>mn(n+1),mx(n+1);
    vector<ll>p(n+1);
    auto dfs = [&](auto& self,int u,int fa) -> void{
        mn[u] = mx[u] = a[u];
        p[u] = 1;
        vector<pair<int,int>>son;
        int k = 0;
        for(int v:g[u]){
            if(v == fa)continue;
            self(self,v,u);
            k++;
            if(p[v] == 0)p[u] = 0;
            son.push_back({mn[v],mx[v]});
            if(p[v] != 0){
                p[u] = p[u]*p[v]%mod;
            }
        }
        if(k == 0||p[u] == 0)return;
        sort(son.begin(),son.end());
        if(a[u] > son[0].first){
            p[u] = 0;
            return;
        }
        for(int i = 0;i < k-1;i++){
            if(son[i].second > son[i+1].first){
                p[u] = 0;
                return;
            }
        }
        ll cnt = 1;
        ll sum = 1;
        for(int i = 0;i < k-1;i++){
            if(son[i] == son[i+1]){
                cnt++;
                sum = sum*cnt%mod;
            }
            else{
                cnt = 1;
            }
        }
        p[u] = p[u]*sum%mod*invf[k]%mod;
        mx[u] = son.back().second;
    };
    dfs(dfs,1,0);
    cout << p[1] << endl;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr); 
    int t;
    cin >> t;
    ff();
    while(t--)solve();
    return 0;
}

```

#### 牛客小苯的路径前缀

`[G-小苯的路径前缀_“⌬杯”蓝桥杯大赛省赛模拟赛](https://ac.nowcoder.com/acm/contest/130222/G)`

题目简述

- 给定树和排列 p[1..n]。
- 求最大的 k，使得前 k 个点能被某条**简单路径**覆盖（路径可以包含未选点）。

核心性质（必须记住）

> 点集能被一条简单路径覆盖 ⇔ 这些点的**极小连通子树（Steiner树）**中，所有节点度数 ≤2（即该子树是一条链，无三分叉）。

**等价判定（DFS 方法）**  
- 从根（代码中固定为 1）DFS。
- 对每个点，统计**含标记点的方向数**：
  - 每个含标记点的**儿子子树**算一个方向。
  - **自己若是标记点**，也贡献一个方向（向父亲方向）。
- 若某节点方向数 ≥3，则必然不存在这样的路径。

代码整体思路

```cpp
// 1. 建边
// 2. 预处理 f 数组（父亲），用于后续 dfs2 屏蔽祖先方向
// 3. 定义 check(mid) 判断前 mid 个点是否可行
// 4. 二分答案 l=1, r=n，用 while(l<=r) 求最大 mid
```

**重要**：你使用 `f[u] = fa` 建树，`dfs(dfs,1,0)`，这是正确的，不能直接在输入时赋值。

---

 `check(mid)` 详细过程

 标记前 mid 个点

```cpp
vector<int> vis(n+1, 0);
for(int i=1; i<=mid; i++) vis[a[i]] = 1;
```
- `vis[x] == 1` 表示点 x 在点集 S_k 内。

第一遍 DFS（判定结构）

```cpp
int lca = 0;
auto dfs = [&](auto self, int u, int fa) -> int {
    if(lca != 0) return lca;       // 已经确定结果，快速退出
    int son = 0;                    // 含标记点的儿子分支数
    int val = 0;                    // 其中一个含标记点的儿子返回的值
    for(auto v : g[u]) {
        if(v == fa) continue;
        int cur = self(self, v, u);
        if(lca != 0) return lca;   // 剪枝
        if(cur > 0) {
            son++;
            val = cur;             // 保存其中一个返回值
        }
    }
    // 下面按 son 数量分类处理:
    if(son == 0) {
        if(vis[u]) return u;
        return 0;
    } else if(son == 1) {
        if(vis[u]) return u;
        return val;
    } else if(son > 2) {
        lca = -1;
        return -1;
    }
    // son == 2 时，到达这里
    lca = u;
    return lca;
};
dfs(dfs, 1, 0);
```

**关键变量解释**
- `son`：当前点 u 的几个儿子子树内含有标记点。
- `val`：仅当 `son==1` 且 `vis[u]==0` 时有用，用于向上传递标记点信息。  
  你之前问过“能不能不用 val”，**可以**。这里保留只是惯用写法，不影响正确性。
- `lca`：这里不是真的最近公共祖先，它在代码里代表**唯一可能的汇合点（分叉点）**。  
  - `0`：尚未出现需要分叉的情况（所有标记点在一条直线上）  
  - `>0`：出现唯一分叉点，编号就是 `lca`  
  - `-1`：已确定不可能（出现 son>2 或两个分叉点）

**特别注意的细节**
- `son == 1 && vis[u] == 1` 时，你执行了 `return u;`  
  这实际上将 u 自身当作了一个**合并点**。因为此时有一个儿子方向 + 自己标记（另一个方向），形成分叉，**且这个分叉点就是 u**。  
  但在你的代码里，这种情况**并没有显式地设置 `lca = u`**，而只是返回了 u。  
  之后如果 u 的祖先还有标记点，会在更高层形成新的分叉，那时才设置 lca。  
  所以只有当 `son == 2`（两个儿子分支都有点）时才显式 `lca = u`。  
  这两种情形本质相同，只是代码选择了在 `son==2` 时才记录 lca。
  这也是为什么 `son==1 && vis[u]` 时不需要额外检查祖先方向，因为此时 u 本身就是一个端点，上方不能再有点（否则会在更高层形成新分叉而被捕获）。

- 在 `son > 2` 时设置 `lca = -1` 直接失败。

 处理 DFS 结果

```cpp
if(lca == 0) return true;     // 无分叉点，成链
if(lca == -1) return false;   // 已经失败
// lca > 0，有唯一分叉点，需检查祖先方向
int cnt = 0;
auto dfs2 = [&](auto& self, int u, int fa) -> void {
    cnt += vis[u];
    for(auto v : g[u]) {
        if(v == fa) continue;
        self(self, v, u);
    }
};
dfs2(dfs2, lca, f[lca]);      // 关键：父节点传入 f[lca]，屏蔽上方
return cnt == mid;
```

**为什么需要 dfs2？** 
当 `son == 2` 在 u 发生时，我们只确认了 u 的**儿子分支**中有标记点，但**没考虑 u 的祖先方向**是否也有标记点。  

- 如果祖先方向还有标记点，那么 u 的方向数实际为 3，就会失败。  
- 通过从 lca 向子树（避开父方向）统计标记点数，如果数目等于 mid，说明全部点都在子树内，祖先方向无点，可行；否则不可行。

**传入 `f[lca]` 的作用** 
`f[lca]` 是 lca 的父亲，`dfs2(dfs2, lca, f[lca])` 会使得递归不会向根方向走，只统计以 lca 为根的子树部分。

---

 二分答案

```cpp
int l = 1, r = n, ans = 0;
while(l <= r) {
    int mid = (l + r) >> 1;
    if(check(mid)) {
        ans = mid;
        l = mid + 1;
    } else {
        r = mid - 1;
    }
}
cout << ans << endl;
```
- 闭区间写法，每次成功就尝试更大值，失败则减小。
- 注意 `ans` 初始化为 0，当全部失败时会输出 0（虽然题目 k≥1，但这样写安全）。

---

易错点（亲测易掉坑）

1. **多组数据**：每次 solve 前 vector 重新分配，没有问题。
2. **vis 数组**：每次 check 都会新建并初始化前 mid 个，没问题。
3. **lca 状态的正确处理**：`lca == -1` 优先判断，`lca == 0` 直接 true，`lca > 0` 走 dfs2。
4. **`son==1` 且 `vis[u]==1` 时**，你的代码没有显式设 lca，但这是安全的，因为此时 u 相当于端点，上层不再允许出现标记点，若出现会在更高层被捕获。
5. **二分边界与初始化**：养成 `ans=0` 的习惯，防止无解时出错。
6. **dfs2 中 `fa` 参数的传递**：必须用 `f[lca]` 作为父节点传入，才能屏蔽祖先。

---

时间与空间复杂度

- 单次 `check`：O(n)（两次 DFS 仅遍历部分节点，整体 O(n)）
- 二分次数：O(log n)
- 总复杂度：O(n log n) 每组，所有测试点 n 总和 ≤ 2e5，可过。
- 空间：O(n)（邻接表、辅助数组）

一句话总结

**判断点集是否在树的一条简单路径上 ⇒ DFS 统计每个点含标记点的方向数，若出现 ≥3 方向则否，唯一分叉点需补检祖先方向**。

**简单路径其实就是看看是否能连成一条链，一般就是找lca，然后看看它们是否符合链的结构。**

**AC代码：**

```c++
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

void solve() {
    int n;
    cin >> n;
    vector<vector<int>>g(n+1);
    int x,y;
    for(int i = 1;i < n;i++){
        cin >> x >> y;
        g[x].push_back(y);
        g[y].push_back(x);
    }
    vector<int>a(n+1,0);
    for(int i = 1;i <= n;i++){
        cin >> a[i];
    }
    vector<int>f(n+1,0);
    auto dfs = [&](auto&self,int u,int fa)->void{
        f[u] = fa;
        for(auto v:g[u]){
            if(v == fa)continue;
            self(self,v,u);
        }
    };
    dfs(dfs,1,0);
    auto check = [&](int mid) ->bool{
        vector<int>vis(n+1,0);
        for(int i = 1;i <= mid;i++){
            vis[a[i]] = 1;
        }
        int lca = 0;
        auto dfs = [&](auto self,int u,int fa)->int{
            if(lca != 0)return lca;
            int son = 0;
            int val = 0;
            for(auto v:g[u]){
                if(v == fa)continue;
                int cur = self(self,v,u);
                if(lca != 0)return lca;
                if(cur > 0){
                    son++;
                    val = cur;
                }
            }
            if(son == 0){
                if(vis[u])return u;
                return 0;
            }
            else if(son == 1){
                if(vis[u])return u;
                return val;
            }
            else if(son > 2){
                lca = -1;
                return -1;
            }
            lca = u;
            return lca;
        };
        dfs(dfs,1,0);
        if(lca == 0)return true;
        if(lca == -1)return false;
        int cnt = 0;
        auto dfs2 = [&](auto&self,int u,int fa)->void{
            cnt += vis[u];
            for(auto v:g[u]){
                if(v == fa)continue;
                self(self,v,u);
            }
        };
        dfs2(dfs2,lca,f[lca]);
        return cnt == mid;
    };

    int l = 1,r = n;
    int ans = 0;
    while(l <= r){
        int mid = (l+r)>>1;
        if(check(mid)){
            ans = mid;
            l = mid+1;
        }
        else{
            r = mid-1;
        }
    }
    cout << ans << endl;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr); 
    int t;
    cin >> t;
    while(t--)solve();
    return 0;
}
```

## 数学

### 组合数学

#### atc457G

问题重述

- 有 \(N\) 个苹果，第 \(i\) 个在时刻 \(T_i\) 落在坐标 \(X_i\)。
- 机器人从时刻 \(0\) 开始可放在任意位置，速度上限为 \(1\)。
- 一个机器人可以收集多个苹果，前提是能在时间与速度的限制下依次到达。
- 求最少需要多少个机器人才能收集所有苹果。

---

核心转化流程图

```
原问题 (T, X)
   ↓ 速度限制 |Xj - Xi| ≤ Tj - Ti
拆绝对值
   ↓ 得到两个线性不等式
引入新坐标:  U = T + X,  V = T - X
   ↓ 条件等价于: Ui ≤ Uj 且 Vi ≤ Vj
二维偏序 (U, V)
   ↓ 按 V 升序排序
求 U 序列的最长严格下降子序列 (LDS)
   ↓ Dilworth 定理保证
答案 = LDS 长度
```

---

详细推导步骤

写出同一机器人连续收集的条件

假设机器人先收苹果 \(i\)，再收苹果 \(j\)（\(T_i \le T_j\)），需满足：
\[
\frac{|X_j - X_i|}{T_j - T_i} \le 1 \quad\Rightarrow\quad |X_j - X_i| \le T_j - T_i
\]

拆绝对值，得到两个独立不等式
\[
\begin{cases}
X_j - X_i \le T_j - T_i \\
-(X_j - X_i) \le T_j - T_i
\end{cases}
\Rightarrow
\begin{cases}
T_i - X_i \le T_j - X_j \\
T_i + X_i \le T_j + X_j
\end{cases}
\]

定义新坐标 (V, U)

令：
\[
V = T - X,\quad U = T + X
\]
则条件变为简洁的二维偏序：
\[
V_i \le V_j \quad\text{且}\quad U_i \le U_j
\]
即苹果 \(i\) 能排在苹果 \(j\) 之前被同一机器人收集。

转化为求最小链覆盖

- 每个苹果是平面点 \((V, U)\)。
- 一个机器人的收集序列是一条 **V, U 均不降的路径**（链）。
- 目标：用最少的这种路径（链）覆盖所有点。
- 根据 **Dilworth 定理**：最小链覆盖数 = 最大反链大小。

求最大反链（最长下降子序列）

将苹果按 **V 升序**排序（V 相同则按 U 升序）。
排序后，任意两个苹果 \(i < j\)（数组顺序）自动满足 \(V_i \le V_j\)。
它们不能同机器人（属于反链）的条件只剩：**U 不满足不降**，即 **\(U_i > U_j\)**。

因此，**反链 = U 序列的严格下降子序列**。  
最大反链大小 = **最长严格下降子序列 (LDS) 的长度**。

答案 = LDS 长度。

---

算法实现：求 LDS 的 O(N log N) 方法

LDS 可以通过对值取负转化为 **最长严格上升子序列 (LIS)** 来求。

LIS 的贪心+二分算法

维护一个数组 `tails`，`tails[k]` 表示当前找到的长度为 `k+1` 的上升子序列的最小可能末尾。
- 遍历每个数 `x`：
  - 用 `lower_bound` 找到第一个 `≥ x` 的位置 `it`（严格上升）。
  - 若不存在，`push_back(x)`（形成更长的子序列）。
  - 否则，`*it = x`（更新该长度下的最小末尾，保持潜力）。

最后 `tails.size()` 即 LIS 长度。

求 LDS 的代码片段

```cpp
vector<int> lis;
for (auto &p : a) {
    int val = -p.second;   // U 取负，严格下降变严格上升
    auto it = lower_bound(lis.begin(), lis.end(), val);
    if (it == lis.end()) lis.push_back(val);
    else *it = val;
}
cout << lis.size() << '\n';   // 最少机器人数
```

---

易错点汇总

| 错误                                     | 正确做法                                                     |
| ---------------------------------------- | ------------------------------------------------------------ |
| 直接数“相邻下降次数+1”作为答案           | 必须求**全局最长下降子序列**长度。相邻下降段数可能高估。     |
| 排序时未处理 V 相同的情况                | 必须按 `(V, U)` 双关键字升序排序，否则可能破坏偏序关系。     |
| 更新 LIS 时写成 `v = *it` 而非 `*it = v` | 导致数组未更新，答案错误。                                   |
| 使用 `upper_bound` 求严格上升子序列      | 严格上升应用 `lower_bound`（找第一个 `≥`），非严格用 `upper_bound`。 |

---

 重要定理回顾

 Dilworth 定理

> 在有限偏序集中，最小链覆盖数 = 最大反链大小。

- **链**：两两可比的元素序列（本题中为一个机器人的收集序列）。
- **反链**：两两不可比的元素集合（互相冲突、不能同机器人的苹果）。
- **应用**：只要把问题转化为二维偏序的最小链覆盖，就能通过求最大反链来得到答案。

LIS 贪心算法正确性

`tails` 始终保持递增，维护“每个长度下的最小可能末尾”，从而保证后续可以尽可能地延长序列。最终长度即为 LIS 长度。

完整代码模板

```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;

void solve() {
    int n;
    cin >> n;
    vector<pair<int, int>> a;
    for (int i = 0; i < n; i++) {
        int T, X;
        cin >> T >> X;
        a.push_back({T - X, T + X}); // (V, U)
    }
    sort(a.begin(), a.end()); // 先V后U升序

    vector<int> lis;
    for (auto &p : a) {
        int val = -p.second; // U取负求LIS
        auto it = lower_bound(lis.begin(), lis.end(), val);
        if (it == lis.end()) lis.push_back(val);
        else *it = val;
    }
    cout << lis.size() << '\n';
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    solve();
    return 0;
}
```

举一反三：这类问题的通用套路

遇到 **“时间+位置，速度限制，最少资源”** 的题目，按以下步骤思考：
1. **写速度约束**：\(|X_j - X_i| \le T_j - T_i\)
2. **拆绝对值引入新变量**：\(U = T+X\)，\(V = T-X\)
3. **得到二维偏序**：\(U_i \le U_j\) 且 \(V_i \le V_j\)
4. **套用 Dilworth 定理**：最少资源 = 最大反链 = 按一维排序后另一维的 LDS
5. **用 LIS 二分法求 LDS**，复杂度 \(O(N \log N)\)
